# Чек-лист индексации

## Перед отправкой в поиск

1. Проверить, что сайт открывается по адресу `https://deputat36.github.io/bm/`.
2. Проверить `robots.txt`.
3. Проверить `sitemap.xml`.
4. Проверить дополнительные карты:
   - `sitemap-extra.xml`;
   - `sitemap-extra-2.xml`.
5. Убедиться, что страницы не отдают 404.
6. Проверить, что на страницах есть уникальные `title` и `description`.

## Яндекс.Вебмастер

1. Добавить сайт.
2. Подтвердить права доступным способом.
3. Отправить `sitemap.xml`.
4. Отправить `sitemap-extra.xml`.
5. Отправить `sitemap-extra-2.xml`.
6. Проверить раздел `Индексирование -> Страницы в поиске`.
7. Проверить ошибки обхода.

## Google Search Console

1. Добавить ресурс.
2. Подтвердить права.
3. Отправить карты сайта.
4. Проверить статус отправленных URL.
5. Проверить проблемы мобильной версии.

## Приоритет ручной отправки URL

В первую очередь отправлять:

1. Главная.
2. `/kvartiry/`.
3. `/ceny/`.
4. `/planirovki/`.
5. `/ipoteka/`.
6. `/investoram/`.
7. `/semeyam/`.
8. `/pokupka-s-prodazhey/`.
9. `/dokumenty/`.
10. Самые свежие статьи из `/novosti/`.

## Что обновлять после новых данных

После появления цен, планировок, документов или фото стройки:

1. Обновить соответствующую страницу.
2. Добавить дату обновления в текст.
3. Проверить внутренние ссылки.
4. Обновить `sitemap.xml`.
5. Отправить изменённый URL на переобход.
