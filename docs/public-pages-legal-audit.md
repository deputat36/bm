# Аудит публичных страниц по юридически безопасным формулировкам

Дата фиксации: 2026-07-07

Документ фиксирует текущий этап приведения публичных страниц сайта к безопасной модели портала «Новостройки Борисоглебска».

## Цель аудита

Снизить риск того, что сайт будет восприниматься как официальный сайт застройщика, официальный сайт ЖК или страница с гарантированной продажей, бронью, фиксацией цены или одобрением ипотеки.

Базовая модель:

```text
Сайт является независимым информационным порталом. Информация носит справочный характер. Заявка с сайта является обращением за консультацией и не является бронированием квартиры, офертой, договором, фиксацией цены или подтверждением наличия квартиры.
```

## Что не удалось изменить напрямую

### `/kvartiry-ot-zastroyschika-borisoglebsk/`

Файл несколько раз блокировался инструментом при полной замене содержимого.

Текущий статус риска:

- страница уже имеет `noindex, follow`;
- canonical указывает на `/kvartiry/`;
- `robots.txt` дополнительно закрывает URL `/kvartiry-ot-zastroyschika-borisoglebsk/` от индексации;
- страница не включена в `sitemap.xml`.

Рекомендация:

1. Удалить эту страницу после проверки, что на нее нет рекламных ссылок.
2. Либо заменить ее на редирект на `/kvartiry/` через серверную конфигурацию.
3. Не использовать формулировку «квартиры от застройщика» в публичной структуре портала без отдельного юридического согласования.

### `/karta-sayta/`

Страница содержит ссылку на `/kvartiry-ot-zastroyschika-borisoglebsk/`. Попытки полной замены карты сайта блокировались инструментом.

Рекомендация:

- удалить ссылку на старую noindex-страницу при следующем техническом редактировании;
- использовать ссылку на `/kvartiry/` как основную страницу квартир;
- в дальнейшем генерировать карту сайта автоматически из безопасного списка URL.

## Текущие контрольные правила для новых страниц

При добавлении новых страниц использовать следующие правила:

1. Не писать «официальный сайт», «официальная страница», «официальный отдел продаж».
2. Не писать «мы — застройщик», «наш ЖК», «наши квартиры от застройщика».
3. Не обещать цену, наличие квартиры, бронь, скидку, доходность или одобрение ипотеки.
4. Указывать источник данных и дату обновления, если страница содержит фактические сведения.
5. В формах использовать только консультационную логику.
6. В футерах давать ссылки на `/legal/`, `/sources/`, `/privacy/`.
7. Для рекламных материалов отдельно проверять маркировку, ОРД и требования к рекламе долевого строительства.

## Следующие задачи

1. Удалить или заменить `/kvartiry-ot-zastroyschika-borisoglebsk/`.
2. Обновить `/karta-sayta/`, убрав ссылку на старую noindex-страницу.
3. Вынести общий header/footer в шаблон или JS-include, чтобы правовой футер был единым на всех страницах.
4. Создать безопасный список URL для будущей генерации sitemap.
5. Подготовить структуру будущего портала `/zhk/tellermanov-sad/`.
6. Начать миграцию текущих страниц ЖК в раздел общего портала.
