# Статическая проверка сайта

Дата фиксации: 2026-07-05

## 1. Назначение

Перед переносом черновой главной `/portal-preview/` в корневой `index.html` нужно проверять:

- локальные ссылки;
- подключения CSS и JS;
- наличие `noindex,follow` на черновых страницах;
- обязательные data-атрибуты форм заявок.

Для этого добавлен валидатор:

```text
tools/validate-static-site.mjs
```

## 2. Запуск

Из корня проекта:

```bash
npm run validate
```

Скрипт не требует внешних зависимостей. Достаточно Node.js.

## 3. Что проверяет валидатор

### Локальные ссылки

Проверяются `href` и `src`, если они не являются:

- внешней ссылкой `http://` или `https://`;
- телефоном `tel:`;
- email `mailto:`;
- якорем `#...`;
- data-url;
- JavaScript-ссылкой.

Если ссылка ведёт на папку, валидатор ожидает `index.html` внутри этой папки.

### Черновые страницы

Для черновых разделов проверяется наличие:

```html
<meta name="robots" content="noindex,follow">
```

Проверяемые черновые разделы:

```text
/portal-preview/
/novostroyki/
/zhk/
/zastroyschiki/
/spravochnik/
/sravnenie/
```

### Формы заявок

Для каждой формы с `data-lead-form` проверяются обязательные атрибуты:

```text
data-lead-type
data-form-id
data-project
data-project-id
data-project-name
```

Для заявок типа `project_consultation` дополнительно нужен:

```text
data-complex-id
```

## 4. Как использовать результат

Если валидатор нашёл ошибки, их нужно исправить до:

- снятия `noindex`;
- переноса `/portal-preview/` в корневой `index.html`;
- обновления sitemap;
- запуска рекламы;
- подключения активной CRM-обработки заявок.

## 5. Ограничения

Валидатор не заменяет ручную проверку в браузере. Он не проверяет:

- визуальное отображение;
- адаптивность;
- фактическую отправку Web3Forms;
- работу Supabase Edge Function;
- корректность данных в Яндекс.Метрике / GA4.

Для этого используется отдельный чек-лист:

```text
docs/link-path-audit.md
docs/lead-launch-checklist.md
```
