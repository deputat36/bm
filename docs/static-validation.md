# Статическая проверка сайта

Дата фиксации: 2026-07-07

## 1. Назначение

Перед переносом черновой главной `/portal-preview/` в корневой `index.html` нужно проверять:

- локальные ссылки;
- подключения CSS и JS;
- наличие `noindex,follow` на черновых страницах;
- критичные и рекомендованные data-атрибуты форм заявок;
- корректность JSON-данных по объектам;
- готовность объекта к публичной публикации.

Для этого добавлен валидатор:

```text
tools/validate-static-site.mjs
```

## 2. Локальный запуск

Из корня проекта:

```bash
npm run validate
```

Скрипт не требует внешних зависимостей. Достаточно Node.js.

## 3. GitHub Actions

Добавлен workflow:

```text
.github/workflows/validate-static-site.yml
```

Он запускается:

- вручную через `workflow_dispatch`;
- на pull request.

Автоматический запуск на каждый push в `main` пока отключён, чтобы новый валидатор не блокировал автономные коммиты до первой ручной проверки всего старого сайта.

## 4. Что проверяет валидатор

### Локальные ссылки

Проверяются `href` и `src`, если они не являются:

- внешней ссылкой `http://` или `https://`;
- телефоном `tel:`;
- email `mailto:`;
- якорем `#...`;
- data-url;
- JavaScript-ссылкой.

Если ссылка ведёт на папку, валидатор ожидает `index.html` внутри этой папки.

### Черновые страницы

Для черновых разделов проверяется наличие:

```html
<meta name="robots" content="noindex,follow">
```

Проверяемые черновые разделы:

```text
/portal-preview/
/novostroyki/
/zhk/
/zastroyschiki/
/spravochnik/
/sravnenie/
```

### Формы заявок

Для каждой формы с `data-lead-form` проверяются критичные атрибуты:

```text
data-lead-type
data-form-id
```

В новых черновых разделах отсутствие этих атрибутов считается ошибкой. На старых опубликованных страницах переходного контура это пока считается предупреждением, чтобы валидатор не блокировал работу над порталом из-за наследованных HTML-страниц. Такие страницы нужно постепенно обновить до общего стандарта перед миграцией.

Как предупреждения проверяются рекомендованные атрибуты:

```text
data-project
data-project-id
data-project-name
```

Если эти рекомендованные атрибуты отсутствуют, `main.js` подставит значения из `SITE_CONFIG`. Но для новых страниц портала лучше задавать их явно.

Для заявок типа `project_consultation` дополнительно нужен обязательный атрибут:

```text
data-complex-id
```

В новых черновых разделах отсутствие `data-complex-id` у `project_consultation` считается ошибкой. На старых опубликованных страницах — предупреждением до полного выравнивания форм.

### JSON-данные проектов

Валидатор проверяет:

```text
data/projects/index.json
data/projects/{slug}.json
data/research/newbuilds-borisoglebsk-2018.json
```

Для объектов проверяются поля:

```text
verification_status
is_public_ready
last_checked_at
```

Поддерживаемые статусы:

```text
confirmed
partially_confirmed
requires_check
do_not_publish
```

Если объект отмечен как `is_public_ready = true`, он обязательно должен иметь `verification_status = confirmed`.

### Согласованность индекса и карточки

Для каждого объекта из `data/projects/index.json` валидатор открывает подробную карточку из поля `data_file` и сверяет ключевые поля:

```text
id
slug
verification_status
is_public_ready
```

Если в индексе и карточке разные значения, это считается ошибкой.

## 5. Как использовать результат

Если валидатор нашёл ошибки, их нужно исправить до:

- снятия `noindex`;
- переноса `/portal-preview/` в корневой `index.html`;
- обновления sitemap;
- запуска рекламы;
- подключения активной CRM-обработки заявок.

Предупреждения не блокируют проверку, но показывают места, которые лучше привести к новому стандарту.

## 6. Ограничения

Валидатор не заменяет ручную проверку в браузере. Он не проверяет:

- визуальное отображение;
- адаптивность;
- фактическую отправку Web3Forms;
- работу Supabase Edge Function;
- корректность данных в Яндекс.Метрике / GA4;
- фактическую актуальность сведений в ЕИСЖС / наш.дом.рф.

Для этого используются отдельные чек-листы и регламенты:

```text
docs/link-path-audit.md
docs/lead-launch-checklist.md
docs/data-publication-rules.md
```
