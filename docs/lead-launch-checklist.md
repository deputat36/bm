# Чек-лист запуска лидогенерации

Документ нужен перед запуском рекламы портала «Новостройки Борисоглебска».

## 1. Проверка форм

Проверить каждую форму на `/zayavka/`:

- заявка по ЖК;
- ипотека;
- подбор квартиры;
- консультация;
- звонок;
- список ожидания.

Для каждой формы проверить:

- обязательные поля;
- чекбокс согласия;
- отправку заявки;
- редирект на `/spasibo/`;
- отображение типа заявки на странице благодарности;
- приход заявки в email;
- запись в Supabase после подключения endpoint;
- уведомление менеджеру после подключения уведомлений.

## 2. Проверка UTM

Открыть тестовую ссылку:

```text
/zayavka/?utm_source=test&utm_medium=check&utm_campaign=lead_launch&utm_content=form_test&realtor=tester#jk
```

После отправки проверить, что в заявке есть:

- `utm_source=test`;
- `utm_medium=check`;
- `utm_campaign=lead_launch`;
- `utm_content=form_test`;
- `realtor=tester`;
- первое касание;
- последнее касание;
- страница отправки.

## 3. Проверка страницы спасибо

После отправки пользователь должен попасть на URL вида:

```text
/spasibo/?type=complex_interest&id=NB-...&status=warm
```

Проверить:

- тип заявки отображается по-человечески;
- ID фиксации виден;
- статус отображается;
- есть кнопка возврата на главную;
- есть кнопка повторной заявки;
- есть кнопка звонка.

## 4. Проверка аналитики

Нужно проверить события:

```text
lead_submit
lead_thankyou_view
```

Где проверять:

- консоль браузера / dataLayer;
- Яндекс.Метрика после установки счётчика;
- GA4 после установки тега;
- рекламные кабинеты после настройки целей.

## 5. Проверка Supabase

После подключения endpoint проверить:

- заявка появилась в `newbuild_leads`;
- событие появилось в `newbuild_lead_events`;
- rate limit пишет попытки в `newbuild_lead_rate_limits`;
- RLS включён;
- прямой публичный доступ к таблицам невозможен;
- функция работает через service role только на сервере.

## 6. Проверка уведомлений

Проверить, что менеджер получает уведомление с данными:

- тип заявки;
- имя;
- телефон;
- ЖК;
- интерес;
- бюджет;
- способ покупки;
- квалификация;
- источник;
- страница заявки.

## 7. Проверка юридической безопасности

На формах и странице спасибо не должно быть формулировок:

- «забронировать квартиру»;
- «зафиксировать цену»;
- «официальный сайт застройщика»;
- «гарантированное одобрение»;
- «точная цена», если цена не подтверждена.

Безопасные формулировки:

- «оставить заявку»;
- «зафиксировать обращение»;
- «добавиться в список ожидания»;
- «получить консультацию»;
- «получить информацию после подтверждения».

## 8. Проверка менеджера

Перед запуском рекламы менеджер должен знать:

- где смотреть заявки;
- что означает `hot`, `warm`, `cold`;
- как быстро звонить;
- как фиксировать результат;
- какие статусы ставить;
- какие формулировки нельзя использовать.

Основной регламент:

```text
docs/manager-lead-handling.md
```

## 9. Минимальный тест перед рекламой

Перед запуском сделать 6 тестовых заявок:

1. ЖК — `complex_interest`.
2. Ипотека — `mortgage`.
3. Подбор — `apartment_selection`.
4. Консультация — `consultation`.
5. Звонок — `callback`.
6. Список ожидания — `waitlist`.

После теста проверить, что все 6 заявок:

- дошли;
- имеют разные типы;
- имеют UTM;
- имеют ID фиксации;
- имеют статус квалификации;
- не нарушают юридические формулировки.
