# Архитектура портала «Новостройки Борисоглебска»

Дата фиксации: 2026-07-05

## 1. Цель проекта

Создать городской портал по всем новостройкам Борисоглебска, где каждая новостройка получает отдельную карточку и набор посадочных страниц, а ЖК «Теллерманов сад» становится первым полноценным разделом внутри портала.

Портал должен решать три задачи:

1. Собирать поисковый спрос по новостройкам Борисоглебска, конкретным адресам, застройщикам, планировкам, ипотеке и ценам.
2. Давать покупателю понятную структуру выбора: сравнение объектов, квартир, сроков, способов покупки и документов.
3. Передавать заявки в работу специалистам: общие заявки по порталу и отдельные заявки по конкретному ЖК.

## 2. Текущее состояние репозитория

Сейчас репозиторий `deputat36/bm` содержит статический сайт, фактически ориентированный на один объект — ЖК «Теллерманов сад» / дом на Просторной 4А.

Ключевые признаки текущего состояния:

- `README.md` описывает проект как статический сайт для GitHub Pages о новом доме на Просторной 4А в Борисоглебске.
- Главная страница `index.html` уже имеет SEO-мета под `tellermanovsad.ru`, заголовок «ЖК Теллерманов сад Борисоглебск — квартиры на Просторной 4А», разделы о ЖК, квартирах, планировках, ценах, ипотеке, галерее, ходе строительства и FAQ.
- В `sitemap.xml` сейчас перечислены URL домена `https://tellermanovsad.ru/`, включая страницы `o-zhk`, `kvartiry`, `planirovki`, `ceny`, `dokumenty`, `galereya`, `zastroyschik`, `ipoteka`, `hod-stroitelstva`, `novosti`, `faq`, `contacts`, `privacy`.
- Форма заявок работает через `assets/js/main.js`: собирает имя, телефон, интерес, комментарий, UTM-метки, `realtor`, `realtor_id`, страницу, заголовок страницы и отправляет заявку через Web3Forms или кастомный endpoint.
- В `robots.txt` уже закрыты служебные и экспериментальные разделы: `docs`, `data`, `tools`, UTM-страницы и часть рекламных посадочных страниц.

## 3. Целевая концепция

Новая структура должна быть не «сайт одного ЖК», а городской портал:

- главный бренд: «Новостройки Борисоглебска»;
- ЖК «Теллерманов сад» — отдельный раздел внутри портала;
- текущий домен `tellermanovsad.ru` можно сохранить как проектный домен или настроить редирект на раздел ЖК;
- будущий основной домен портала нужно выбрать отдельно: например, `novostroyki-borisoglebsk.ru`, `novostroyki-bsk.ru`, `bsk-novostroyki.ru` или другой короткий вариант.

### 4.2. Главная страница портала

Главная страница должна стать входом в рынок новостроек Борисоглебска.

Блоки главной:

1. Первый экран: «Новостройки Борисоглебска» + быстрый подбор квартиры.
2. Карточки актуальных ЖК.
3. Отдельный промо-блок ЖК «Теллерманов сад» как первого детально проработанного объекта.
4. Фильтры: район, срок сдачи, тип квартиры, способ покупки, застройщик.
5. Блок «Как купить квартиру в новостройке».
6. Ипотека и господдержка.
7. Новости рынка новостроек Борисоглебска.
8. FAQ.
9. Форма заявки на подбор новостройки.

### 4.3. Каталог новостроек

Раздел `/novostroyki/` должен содержать список объектов:

- название ЖК;
- адрес;
- застройщик;
- статус: анонс, строится, продажи открыты, сдан;
- срок сдачи;
- типы квартир;
- диапазон площадей;
- диапазон цен, когда появятся подтвержденные цены;
- доступные способы покупки;
- ссылка на подробную страницу.

### 4.4. Карточка ЖК

Каждый ЖК должен иметь единую структуру:

```text

/zhk/{slug}/o-zhk/
/zhk/{slug}/kvartiry/
/zhk/{slug}/planirovki/

/zhk/{slug}/galereya/

```

Для ЖК «Теллерманов сад» целевой slug: `tellermanov-sad`.

### 4.5. Справочник покупателя

Раздел `/spravochnik/` нужен для SEO и доверия:

- как проверить новостройку по проектной декларации;
- чем ДДУ отличается от предварительной заявки;
- как работает семейная ипотека;
- как использовать материнский капитал;
- как купить новостройку после продажи своей квартиры;
- какие документы запросить перед покупкой;
- как выбрать планировку;
- как оценить район и инфраструктуру.

## 7. Принципы разработки

1. Не ломать текущие работающие страницы ЖК «Теллерманов сад».
2. Сначала создать архитектурный слой портала, затем переносить страницы.
3. Все изменения делать маленькими коммитами.
4. Сначала документация и карта URL, затем структура папок, затем перенос контента.
5. Все спорные коммерческие утверждения помечать как предварительные, если они еще не подтверждены официально.
6. Служебные разделы `docs`, `data`, `tools` не индексировать.
7. Формы заявок должны сохранять источник, страницу, UTM-метки и интересующий ЖК.

## 8. Ближайший технический курс

1. Создать документацию по архитектуре портала.
2. Создать отдельный план миграции `tellermanovsad.ru` в раздел `/zhk/tellermanov-sad/`.
3. Обновить `README.md`, чтобы он описывал не только дом на Просторной 4А, а будущий портал.
4. Подготовить статическую главную портала.
5. Создать раздел `/zhk/tellermanov-sad/` и начать перенос страниц без удаления старых URL.
6. Добавить карту редиректов.
7. Обновить `sitemap.xml` и `robots.txt` только после готовности новой структуры.
