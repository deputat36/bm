# Полный аудит сайта tellermanovsad.ru

Дата: 29.06.2026
Репозиторий: deputat36/bm
Домен: tellermanovsad.ru

## 1. Итоговая оценка

Сайт уже подготовлен к запуску на собственном домене и содержит актуальные официальные данные по ЖК «Теллерманов сад». Базовая SEO-структура есть: главная, страницы квартир, планировок, цен, документов, застройщика, ипотеки, FAQ, галереи, sitemap и robots.

Главные проблемы перед рекламой:

1. Формы заявок пока не отправляют лиды на сервер.
2. Не подключена аналитика и цели.
3. Есть устаревшая страница `dom-prostornaya-4a`, где встречаются неподтверждённые обещания.
4. Часть внутренних страниц ссылается на старый путь `dom-prostornaya-4a` вместо актуального `/o-zhk/`.
5. На внутренних страницах не везде добавлены canonical и Open Graph.
6. В галерее часть изображений может отображаться заглушками.
7. Нет отдельной страницы политики/согласия на обработку персональных данных.

## 2. Что сделано хорошо

- Домен добавлен через CNAME.
- `robots.txt` указывает на sitemap нового домена.
- `sitemap.xml` переведён на `https://tellermanovsad.ru/`.
- Главная страница содержит canonical, og:url, og:image и JSON-LD.
- Тексты не обещают продажу или бронь, пока продажи на наш.дом.рф не начались.
- В основных разделах используются официальные данные: 70 квартир, площади 27,71–63,76 м², срок сдачи I кв. 2028, ключи до 30.09.2028.
- Есть юридическая оговорка, что сайт информационный и не является официальным сайтом застройщика.

## 3. Критические задачи

### 3.1. Подключить приём заявок

В `assets/js/main.js` поле `LEAD_ENDPOINT` пустое. Сейчас форма сохраняет заявки только в localStorage пользователя и показывает тестовое сообщение.

Нужно подключить один из вариантов:

- Web3Forms;
- Supabase Edge Function;
- собственный обработчик;
- CRM-обработчик.

### 3.2. Исправить старую страницу `dom-prostornaya-4a`

Страница содержит старую информацию и неподтверждённые формулировки: закрытая территория, своя котельная, панорамные окна, предчистовая отделка. До официального подтверждения такие обещания нельзя оставлять на сайте.

Решение: обновить страницу под официальные данные или сделать её технической страницей-переходом на `/o-zhk/`.

### 3.3. Исправить внутренние ссылки

Нужно заменить ссылки `../dom-prostornaya-4a/` на `../o-zhk/` в разделах:

- галерея;
- инфраструктура;
- ход строительства;
- новости.

### 3.4. Добавить согласие на обработку персональных данных

Формы собирают имя и телефон. Нужно добавить:

- страницу `/privacy/`;
- чекбокс согласия в формах;
- ссылку на политику рядом с кнопкой отправки.

## 4. SEO-аудит

### Плюсы

- Главная оптимизирована под запросы: ЖК Теллерманов сад, Борисоглебск, Просторная 4А, квартиры в новостройке.
- Sitemap содержит основные разделы сайта.
- Robots не закрывает публичные страницы.
- Есть отдельная страница проектной декларации.
- Есть страницы под документы, застройщика, квартиры, цены, ипотеку и FAQ.

### Минусы

- На внутренних страницах не хватает canonical.
- На внутренних страницах не хватает og:url и og:image.
- У части страниц слабые title/description: инфраструктура, ход строительства, новости.
- Нет живых новостей и статей.
- Нет страницы политики конфиденциальности.
- Нет favicon.

## 5. Коммерческий аудит

### Сильные стороны

- Понятный первый экран.
- Есть телефон на главной.
- Есть акцент на предварительный список покупателей.
- Есть основные форматы квартир.
- Есть осторожные формулировки по ценам и броням.

### Слабые места

- Нет реальной отправки заявок.
- Нет быстрых мессенджеров.
- Нет блока «кому подойдёт» на главной в более продающем формате.
- Нет понятного сценария для риэлторов и UTM-ссылок.
- Нет калькулятора или заявки на ипотеку отдельным блоком.

## 6. Технический аудит

### Плюсы

- Сайт статический, простой и быстрый.
- Нет тяжёлых фреймворков.
- CSS вынесен в отдельный файл.
- Формы собирают UTM-метки.

### Минусы

- Часть HTML-страниц минифицирована в одну строку, это усложняет поддержку.
- Нет 404.html.
- Нет favicon.
- Нет sitemap lastmod.
- Таблицы могут быть неудобны на мобильных устройствах без горизонтальной прокрутки.

## 7. Контент-аудит

### Нужно добавить

- актуальную страницу хода строительства;
- страницу инфраструктуры с осторожными формулировками;
- 3–5 стартовых новостей;
- блок документов на главной;
- блок «как купить»;
- блок «что подготовить до старта продаж»;
- страницу для сотрудников/риэлторов с готовыми ссылками и формулировками.

## 8. Приоритетный план

### Срочно, до рекламы

1. Подключить обработчик заявок.
2. Подключить Яндекс.Метрику.
3. Исправить старую страницу `dom-prostornaya-4a`.
4. Исправить внутренние ссылки.
5. Добавить privacy и согласие на обработку данных.
6. Проверить сайт на телефоне.

### После первичного запуска

1. Добавить новости.
2. Доработать инфраструктуру.
3. Дозагрузить рендеры.
4. Добавить favicon.
5. Улучшить Open Graph для всех страниц.
6. Добавить 404.html.

## 9. Вывод

Сайт можно показывать как предварительный информационный лендинг, но запускать платную рекламу на формы пока рано. Сначала нужно подключить приём заявок, аналитику, согласие на обработку данных и убрать устаревшие/рискованные формулировки.
