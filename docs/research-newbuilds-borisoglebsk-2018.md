# Исследование новостроек Борисоглебска с 2018 года

Дата фиксации: 2026-07-05

Документ нужен для подготовки наполнения портала `novostroyki-borisoglebsk.ru` и дальнейшего переноса данных в карточки ЖК, страницы застройщиков, таблицу сравнения и справочные материалы.

Важно: это рабочий документ. Он не заменяет юридическую проверку, сверку с ЕИСЖС, проектной декларацией, разрешением на строительство, разрешением на ввод и официальными материалами застройщиков.

## 1. Цель исследования

Собрать по Борисоглебску Воронежской области все многоквартирные жилые дома и проекты нового строительства, которые относятся к периоду с 2018 года по текущую дату:

- построенные и введенные в эксплуатацию дома;
- строящиеся дома;
- планируемые проекты, если есть публичные подтверждения;
- застройщиков и связанные юридические лица;
- адреса, сроки, документы, характеристики и квартирографию;
- степень подтверждения по каждому полю.

Финальная цель — получить структурированный массив данных, который можно безопасно перенести на портал без создания впечатления официального сайта застройщика.

## 2. Что считаем новостройкой для портала

В исследование включаются:

1. Многоквартирные жилые дома, построенные или строящиеся в Борисоглебске с 2018 года.
2. Жилые комплексы или отдельные дома с проектной декларацией.
3. Дома, по которым есть разрешение на строительство, разрешение на ввод, карточка ЕИСЖС или официальная публикация.
4. Планируемые проекты, если есть публичный документ или официальный источник.

В исследование не включаются как подтвержденные объекты:

- частные дома ИЖС;
- устные сведения без публичного подтверждения;
- рекламные объявления без связи с документами;
- планы, которые не подтверждены официальным источником;
- рабочие инсайды, если застройщик просил не публиковать данные.

## 3. Основные источники проверки

Приоритет источников:

1. ЕИСЖС / наш.дом.рф — карточка объекта, проектная декларация, сведения о застройщике, сроки, характеристики.
2. Разрешение на строительство — номер, дата, орган выдачи, адрес, параметры объекта.
3. Разрешение на ввод в эксплуатацию — для построенных домов.
4. Публичные документы администрации, постановления, градостроительные планы, публичные слушания.
5. ЕГРЮЛ / ФНС — проверка юридического лица застройщика.
6. Официальные сайты и страницы застройщиков.
7. Публичные объявления и СМИ — только как вспомогательные источники, не как единственное подтверждение характеристик.

## 4. Статусы проверки

Для каждого объекта и каждого важного поля нужно ставить статус.

### confirmed

Можно переносить в публичную карточку после редакционной проверки.

Условия:

- есть официальный публичный источник;
- указана дата проверки;
- есть ссылка или номер документа;
- данные не противоречат другим источникам.

### partially_confirmed

Можно хранить в рабочей базе, но публиковать осторожно и с оговорками.

Пример:

- подтвержден адрес и застройщик, но не найдена квартирография;
- есть карточка объекта, но нужно сверить актуальную редакцию проектной декларации;
- есть разрешение, но нет подтверждения начала продаж.

### requires_check

Нельзя публиковать как факт.

Пример:

- информация есть в объявлениях, но нет документов;
- данные получены из рабочих разговоров;
- есть старые сведения, но неизвестна актуальность;
- объект упоминается в материалах, но не найден официальный источник.

### do_not_publish

Не переносить на портал.

Пример:

- застройщик или источник просил не публиковать данные;
- сведения коммерчески чувствительные;
- нет публичных подтверждений;
- есть риск создать впечатление официального анонса.

## 8. Как фиксировать построенные дома

Для домов, уже введенных в эксплуатацию, приоритет такой:

1. Найти разрешение на ввод в эксплуатацию.
2. Сопоставить адрес с кадастровыми и публичными сведениями.
3. Найти застройщика или технического заказчика.
4. Проверить, была ли проектная декларация, если дом строился по 214-ФЗ.
5. Отдельно указать, продаются ли еще квартиры от юрлица или объект уже полностью вторичный.

В карточке построенного дома важно не создавать впечатление, что квартиры доступны от застройщика, если это не подтверждено.

## 9. Как фиксировать строящиеся дома

Для строящихся объектов минимум:

- адрес;
- застройщик;
- номер проектной декларации;
- карточка ЕИСЖС;
- разрешение на строительство;
- плановый срок ввода / передачи ключей;
- этажность;
- количество квартир;
- диапазон площадей;
- дата последней проверки.

## 10. Как фиксировать планируемые проекты

Планируемый объект можно добавлять в рабочую базу только если есть публичный источник:

- постановление или документ администрации;
- градостроительная документация;
- официальное сообщение застройщика;
- открытая карточка в ЕИСЖС;
- разрешение на строительство.

Если данные получены из закрытого разговора или застройщик просил не публиковать информацию, объект получает статус `do_not_publish` и не выводится на сайт.

## 11. Формат для переноса на портал

Для портала удобно хранить три слоя данных:

1. `data/research/newbuilds-borisoglebsk-2018.json` — рабочий исследовательский реестр.
2. `data/projects/{slug}.json` — публичные карточки объектов после проверки.
3. `data/builders/{slug}.json` — карточки застройщиков.

Публичные страницы должны брать только объекты со статусом:

```text
verification_status = confirmed
is_active = true
```

Объекты со статусом `requires_check` и `do_not_publish` не должны попадать в публичные карточки, sitemap, SEO-тексты и рекламные материалы.

## 12. Контрольный чек-лист перед публикацией объекта

- [ ] У объекта есть публичный источник.
- [ ] Указана дата последней проверки.
- [ ] Указан статус проверки.
- [ ] Сверены адрес, застройщик и документы.
- [ ] Сверены сроки строительства и передачи ключей.
- [ ] Сверена квартирография.
- [ ] Цены не указаны без актуального подтверждения.
- [ ] Формулировки не создают впечатление официального сайта застройщика.
- [ ] В карточке есть дисклеймер.
- [ ] Объект не нарушает договоренности о непубликации.

## 13. Ближайшие исследовательские задачи

1. Составить первичный список адресов многоквартирных домов, введенных в эксплуатацию в Борисоглебске с 2018 года.
2. Сверить список с ЕИСЖС и документами администрации.
3. Для каждого найденного адреса завести рабочую строку в `data/research/newbuilds-borisoglebsk-2018.json`.
4. Переносить в `data/projects/` только те объекты, где есть публичное подтверждение.
5. Обновлять `/sravnenie/` и `/novostroyki/` только после проверки данных.
