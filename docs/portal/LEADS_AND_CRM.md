# Лиды и CRM портала novostroyki-borisoglebsk.ru

Дата обновления: 2026-07-11

## 1. Коммерческий приоритет

Основной сбор заявок ведётся по трём объектам:

```text
Просторная 4А
Аэродромная 18Г
Сенная 76
```

Остальные новостройки представлены в справочном каталоге. Для них используется общая форма подбора, пока объект не получает отдельный коммерческий приоритет.

## 2. Основные типы заявок

### `project_consultation`

Заявка по конкретному объекту.

Используется на страницах:

```text
/catalog/prostornaya-4a/
/catalog/aerodromnaya-18g/
/catalog/sennaya-76/
```

Обязательные данные:

```text
name
phone
residential_complex
residential_complex_id
lead_type
form_id
project_id
page_url
created_at
personal_data_consent
```

Желательные данные:

```text
interest
budget
purchase_method
timeline
comment
```

### `portal_selection`

Общая заявка на подбор новостройки.

Используется на главной и в каталоге.

Пользователь должен выбрать:

```text
Просторная 4А
Аэродромная 18Г
Сенная 76
Общий подбор новостройки
```

## 3. Идентификаторы приоритетных форм

| Страница | form_id | lead_type | Объект |
|---|---|---|---|
| Главная | `homepage_priority_selection` | `portal_selection` | выбирается пользователем |
| Каталог | `catalog_priority_selection` | `portal_selection` | выбирается пользователем |
| Просторная 4А | `catalog_prostornaya_4a_priority_lead` | `project_consultation` | `prostornaya-4a` |
| Аэродромная 18Г | `catalog_aerodromnaya_18g_priority_lead` | `project_consultation` | `aerodromnaya-18g` |
| Сенная 76 | `catalog_sennaya_76_priority_lead` | `project_consultation` | `sennaya-76` |

## 4. Квалификация заявки

Текущий JavaScript оценивает заявку по следующим признакам:

```text
валидный телефон
понятный интерес
указанный бюджет
способ покупки
срок принятия решения
запрос обратного звонка
ипотечный интерес
встречная продажа
```

Статусы:

```text
hot  — срочно обработать
warm — обработать в рабочий день
cold — добавить в прогрев
```

## 5. Рекомендуемый порядок обработки

### Просторная 4А

1. Связаться максимально быстро.
2. Уточнить комнатность и бюджет.
3. Уточнить способ покупки.
4. Сообщать только подтверждённые цены, наличие, планировки и условия.
5. Зафиксировать следующий контакт.

### Аэродромная 18Г

1. Уточнить, что именно хочет узнать клиент.
2. Не обещать наличие, цену или старт продаж.
3. Добавить клиента в список обновлений по объекту.
4. Предложить сравнение с другими доступными новостройками.

### Сенная 76

1. Уточнить цель обращения.
2. Не публиковать и не сообщать неподтверждённые данные как факт.
3. Зафиксировать интерес к конкретному адресу.
4. Предложить общий подбор, если покупка срочная.

## 7. Страница благодарности

После успешной отправки пользователь переходит на:

```text
/spasibo/
```

Страница должна показывать:

```text
тип заявки
интересующий объект
статус обработки
ID фиксации
```

Название объекта берётся из последней сохранённой заявки в `localStorage`.

## 8. Обязательная проверка форм

Для каждой формы нужно выполнить реальный тест:

1. Открыть страницу без UTM.
2. Отправить тестовую заявку.
3. Проверить получение письма.
4. Проверить объект в теме и содержимом заявки.
5. Проверить `form_id`.
6. Проверить переход на `/spasibo/`.
7. Проверить отображение объекта на странице благодарности.
8. Повторить тест с UTM-метками.
9. Проверить мобильное устройство.
10. Проверить ошибку отправки и резервный телефон.

Тестовые номера и персональные данные нельзя использовать в реальной обработке без пометки `ТЕСТ`.

## 9. Юридические правила

Во всех формах и сообщениях использовать:

```text
заявка
обращение
консультация
предварительный интерес
сообщить после подтверждения
уточнить цену и наличие
```

Не использовать без подтверждённых оснований:

```text
бронь
гарантированная цена
квартира закреплена
официальный отдел продаж
цены от застройщика
официальный представитель
гарантированное одобрение ипотеки
```
