# Атрибуция страницы благодарности

Дата обновления: 2026-07-14

## Цель

Страница `/spasibo/` должна подтверждать отправку обращения и фиксировать завершение воронки без подстановки данных от другой или устаревшей заявки.

## Проблема прежнего сценария

До обновления название объекта бралось из последней записи `newbuildsBorisoglebskLastLead` без проверки, что запись относится к ID заявки из URL.

Это могло привести к неверному отображению объекта, если:

- посетитель отправил несколько заявок в разных вкладках;
- в браузере сохранилась старая запись;
- страницу благодарности открыли вручную;
- локальное хранилище было изменено или очищено.

## Новый порядок определения контекста

1. Страница получает неперсональные параметры `type`, `status` и технический ID заявки.
2. Локальная запись используется только при точном совпадении `client_fixation_id`.
3. `residential_complex_id` проверяется по закрытому реестру допустимых объектов.
4. При отсутствии совпавшей записи допускается ограниченный fallback по same-origin referrer.
5. Если объект определить нельзя, показывается нейтральное значение «Новостройки Борисоглебска».

Допустимые ID:

```text
prostornaya-4a
aerodromnaya-18g
sennaya-76
all-newbuilds
```

Название объекта не выводится напрямую из query string или несверенной записи `localStorage`.

## Аналитика

Событие просмотра:

```text
lead_thankyou_view
```

Поля:

```text
lead_type
form_id
residential_complex
residential_complex_id
qualification_status
attribution_source
```

Технический `client_fixation_id` не передаётся в событии страницы благодарности.

Действия после отправки фиксируются событием:

```text
lead_postsubmit_action
```

Возможные действия:

```text
return_object
catalog
new_lead
phone
phone_header
```

Событие позволяет понять, что посетитель делает после отправки: возвращается к объекту, открывает каталог, создаёт новое обращение или звонит.

## Dry-run

В режиме:

```text
?lead_test=dry-run&test_ack=1
```

или после тестовой отправки с `dry_run=1`:

- не создаётся `lead_thankyou_view`;
- не создаётся `lead_postsubmit_action`;
- страница явно показывает тестовый статус;
- тестовая запись не должна обрабатываться менеджером.

## Автоматическая проверка

Команда:

```bash
npm run validate
```

запускает `tools/validate-thankyou-context.mjs`, который проверяет:

- совпадение технического ID перед чтением локальной записи;
- закрытый список объектов;
- same-origin проверку referrer;
- отсутствие прямого вывода объекта из query string;
- отсутствие `client_fixation_id` в аналитике страницы благодарности;
- отключение аналитики действий в dry-run;
- наличие ссылки возврата к объекту;
- синтаксис встроенного JavaScript.

## Ограничения

Изменение не подтверждает реальную доставку заявки и не заменяет CRM.

Если локальная запись отсутствует, страница не пытается восстановить персональные или подробные данные. Это намеренное безопасное поведение.
