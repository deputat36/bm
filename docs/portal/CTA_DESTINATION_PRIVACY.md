# Приватность назначения CTA

Дата обновления: 2026-07-15

## Цель

Событие `lead_cta_click` должно показывать тип действия, размещение и объект, но не должно передавать в аналитику:

- фактический номер телефона;
- email-адрес;
- произвольные query-параметры;
- fragment после `#`;
- случайные контактные или секретные значения из ссылки.

Исходный `href` используется браузером для навигации и не изменяется. Для аналитики строится отдельное обезличенное представление назначения.

## Телефон

Для ссылок `tel:` и действий `data-track-action="phone"` в поле `link_url` передаётся только:

```text
phone:primary_sales_phone
```

Фактический номер остаётся в HTML-ссылке, необходимой пользователю для звонка, но не экспортируется в `dataLayer`, прямой `gtag`, локальный analytics debug или внутренний `portalConversionEvent`.

Клик по номеру не означает:

- состоявшийся звонок;
- соединение;
- разговор;
- уникального позвонившего;
- заявку.

## Email

Для `mailto:` используется токен:

```text
email:portal_contact
```

Сам email-адрес в событие не передаётся.

## Внутренние ссылки

Для внутреннего URL сохраняются:

```text
pathname + разрешённые параметры атрибуции
```

Разрешены только:

```text
utm_source
utm_medium
utm_campaign
utm_content
utm_term
utm_id
gclid
yclid
ymclid
vkclid
fbclid
roistat
openstat
realtor
realtor_id
manager
lead_source
placement
```

Произвольные параметры, включая `phone`, `email`, `name`, `comment`, `token`, удаляются. Fragment не сохраняется.

Параметр выбора объекта в ипотечном маршруте не нужен в `link_url`, потому что объект уже передаётся отдельным полем `object_id`.

## Внешние ссылки

Для внешнего назначения сохраняются только:

```text
origin + pathname
```

Query-параметры, fragment, логин и пароль не экспортируются.

## Событие

Пример телефонного события:

```json
{
  "event": "lead_cta_click",
  "page_path": "/catalog/prostornaya-4a/",
  "action": "phone",
  "placement": "project_contact",
  "object_id": "prostornaya-4a",
  "link_url": "phone:primary_sales_phone"
}
```

`link_url` не является разрешённым измерением телефонного отчёта. Для анализа используются только:

```text
page_path
placement
object_id
```

## Автоматическая проверка

Проверка выполняется существующим workflow:

```text
Phone CTA guard
```

Validator контролирует:

- наличие `sanitizeCtaDestination`;
- замену `tel:` на `phone:primary_sales_phone`;
- замену `mailto:` на `email:portal_contact`;
- отсутствие прямого `target.getAttribute("href")` в `link_url`;
- очистку внутренних и внешних URL;
- запрет номера и `link_url` как измерений телефонного отчёта;
- соответствие реестру событий;
- разметку телефонных ссылок на семи активных страницах.

## Ограничения

Автоматический guard не заменяет проверку браузера. При ручном QA нужно открыть локальный analytics debug и убедиться, что:

- телефонное событие содержит технический токен;
- фактический номер отсутствует;
- внутренний URL не содержит произвольных параметров;
- переход пользователя по исходной ссылке работает штатно.

Рабочий GA4 DebugView и фактические события внешнего счётчика пока не проверялись.
