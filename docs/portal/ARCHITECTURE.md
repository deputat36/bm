# Архитектура портала novostroyki-borisoglebsk.ru

Дата фиксации: 2026-07-08

Рабочий репозиторий: `deputat36/bm`.

## 1. Целевая идея

`novostroyki-borisoglebsk.ru` — городской портал по новостройкам Борисоглебска.

Главная страница должна быть каталогом всех новостроек, а не страницей одного ЖК.

Главная задача архитектуры — разделить:

- портал;
- каталог объектов;
- карточки конкретных домов и ЖК;
- застройщиков;
- ипотеку;
- справочник;
- новости;
- контакты и формы заявок.

## 2. Целевая структура URL

Базовая структура:

```text
/
/catalog/
/catalog/prostornaya-4a/
/catalog/aerodromnaya-18g/
/catalog/sennaya-76/
/developers/
/ipoteka/
/guides/
/news/
/contacts/
```

## 3. Назначение разделов

### `/`

Главная страница портала.

Должна содержать:

- короткое позиционирование портала;
- каталог актуальных новостроек;
- фильтры или быстрые ссылки по объектам;
- блок сравнения объектов;
- блок ипотеки;
- ссылки на справочник;
- форму общей заявки на подбор квартиры.

Не должна выглядеть как официальный сайт одного застройщика или одного ЖК.

### `/catalog/`

Каталог всех новостроек Борисоглебска.

Должен содержать карточки:

```text
Просторная 4А
Аэродромная 18Г
Сенная 76
```

Для каждой карточки:

- адрес;
- статус;
- застройщик;
- сроки;
- типы квартир;
- документы;
- статус проверки данных;
- кнопка перехода в карточку объекта;
- кнопка заявки.

### `/catalog/prostornaya-4a/`

Карточка объекта Просторная 4А.

Связанный текущий объект: ЖК «Теллерманов сад».

Страница должна быть объектной карточкой каталога, а не отдельным промо-сайтом.

Содержит:

- адрес;
- характеристики;
- застройщика;
- квартирографию;
- документы;
- сроки;
- ипотеку;
- ход строительства;
- FAQ;
- форму заявки по объекту.

### `/catalog/aerodromnaya-18g/`

Карточка объекта Аэродромная 18Г.

На первом этапе можно создать черновик с пометкой:

```text
Данные уточняются.
```

Нельзя публиковать неподтверждённые цены, планировки и документы как факт.

### `/catalog/sennaya-76/`

Карточка объекта Сенная 76.

На первом этапе можно создать черновик с пометкой:

```text
Данные уточняются.
```

Нельзя публиковать неподтверждённые цены, планировки и документы как факт.

### `/developers/`

Раздел застройщиков.

Заменяет старую логику `/zastroyschiki/`.

Должен содержать:

- список застройщиков;
- юридические лица;
- связанные объекты;
- ссылки на карточки объектов;
- статус проверки данных.

### `/ipoteka/`

Раздел по ипотеке и способам покупки.

Содержит:

- обычная ипотека;
- семейная ипотека;
- материнский капитал;
- сертификат «Молодая семья»;
- продажа своей недвижимости;
- предварительная консультация.

Формулировки должны быть аккуратными: заявка не является одобрением банка.

### `/guides/`

Справочник покупателя.

Заменяет старую логику `/spravochnik/`.

Темы:

- как проверить документы новостройки;
- проектная декларация;
- разрешение на строительство;
- как выбрать планировку;
- что проверить перед покупкой;
- как купить квартиру в новостройке.

### `/news/`

Новости портала.

Заменяет старую логику `/novosti/`.

Новости должны быть разделены по типам:

- новости портала;
- новости объектов;
- обновления документов;
- ход строительства;
- ипотечные новости;
- городская инфраструктура рядом с новостройками.

### `/contacts/`

Контакты портала и форма обращения.

Не должна выглядеть как контактная страница одного ЖК.

## 4. Карта текущих URL к новой архитектуре

```text
/                                  → /                       будущая главная портала
/portal-preview/                   → /                       черновик будущей главной
/novostroyki/                      → /catalog/
/zhk/                              → /catalog/
/zhk/tellermanov-sad/              → /catalog/prostornaya-4a/
/zhk/tellermanov-sad/o-zhk/        → /catalog/prostornaya-4a/
/zhk/tellermanov-sad/kvartiry/     → /catalog/prostornaya-4a/
/zhk/tellermanov-sad/planirovki/   → /catalog/prostornaya-4a/
/zhk/tellermanov-sad/ceny/         → /catalog/prostornaya-4a/
/zhk/tellermanov-sad/dokumenty/    → /catalog/prostornaya-4a/
/zastroyschiki/                    → /developers/
/zastroyschiki/bm-group/           → /developers/bm-group/
/spravochnik/                      → /guides/
/novosti/                          → /news/
/contacts/                         → /contacts/
```

До финального решения это не правила редиректов, а карта будущей миграции.

## 5. Слои данных

Рекомендуемая структура данных:

```text
/data/site.json
/data/projects/index.json
/data/projects/prostornaya-4a.json
/data/projects/aerodromnaya-18g.json
/data/projects/sennaya-76.json
/data/developers/index.json
/data/pages/index.json
/data/pages/seo-drafts.json
```

Текущие файлы `/data/builders/` можно сохранить как legacy-слой или постепенно переименовать в `/data/developers/` после проверки зависимостей.

## 6. Модель объекта каталога

Минимальная карточка объекта:

```json
{
  "id": "prostornaya-4a",
  "slug": "prostornaya-4a",
  "name": "Просторная 4А",
  "display_name": "Дом на Просторной 4А",
  "city": "Борисоглебск",
  "address": "ул. Просторная, 4А",
  "developer_id": "bm-group-chernozemye",
  "status": "Данные уточняются",
  "sales_status": "Данные уточняются",
  "detail_url": "/catalog/prostornaya-4a/",
  "is_public_ready": false,
  "verification_status": "draft",
  "is_active": true
}
```

Для объектов с неполными данными использовать:

```text
status: Данные уточняются
is_public_ready: false
verification_status: draft
```

## 7. Статусы страниц

Черновик:

```text
robots: noindex,follow
status: draft
```

Готово к финальному просмотру:

```text
robots: noindex,follow
status: ready
```

Опубликовано:

```text
robots: index,follow
status: published
```

В sitemap можно добавлять только страницы со статусом `published`.

## 8. Формы заявок

Формы должны передавать:

```text
lead_type
form_id
project_id
project_name
residential_complex_id или object_id
source_page
utm_source
utm_medium
utm_campaign
realtor
realtor_id
```

Для общей заявки портала:

```text
lead_type: portal_selection
project_id: newbuilds-borisoglebsk
project_name: Новостройки Борисоглебска
```

Для заявки по конкретному объекту:

```text
lead_type: project_consultation
object_id: prostornaya-4a
```

## 9. SEO-логика

Главная семантика портала:

```text
новостройки Борисоглебска
квартиры в новостройках Борисоглебск
купить квартиру в новостройке Борисоглебск
новые дома Борисоглебск
семейная ипотека новостройка Борисоглебск
```

Объектная семантика должна уходить в `/catalog/{object}/`, а не занимать корень сайта.

## 10. Микроразметка

Для главной:

```text
WebSite
Organization или LocalBusiness — только если юридически корректно
ItemList для каталога объектов
BreadcrumbList
```

Для карточки объекта:

```text
Residence или ApartmentComplex — только при совпадении с видимым содержанием
BreadcrumbList
FAQPage — только для видимого FAQ
```

Не добавлять `Offer` с ценой без подтверждённых цен.

## 11. Что делать первым

1. Создать физическую структуру `/catalog/`, `/developers/`, `/guides/`, `/news/` как черновики.
2. Создать `/catalog/index.html` с карточками трёх объектов.
3. Создать `/catalog/prostornaya-4a/index.html` как новую объектную карточку портала.
4. Создать заглушки `/catalog/aerodromnaya-18g/` и `/catalog/sennaya-76/` с безопасным текстом «данные уточняются».
5. Не трогать текущий `/index.html` до успешной проверки.
6. Не обновлять sitemap/canonical до финального перехода.
