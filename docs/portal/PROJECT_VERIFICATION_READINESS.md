# Готовность объектов к публикации

Дата проверки: 2026-07-16

## Итог

```text
Готово к публичной публикации: 0 из 3
Требует повторной проверки: 3
Требует первичных источников: 0
```

Все три страницы сохраняют noindex,follow.

Статус `requires_recheck` означает, что для объекта создан внутренний verification-профиль, но его critical claims ещё не подтверждены принятыми первичными источниками.

Это разрешает:

- собирать предварительные обращения;
- хранить рабочие гипотезы и вопросы для проверки;
- показывать безопасный статус «Данные уточняются»;
- использовать объект во внутреннем консультационном контуре.

Это не разрешает:

- снимать `noindex`;
- добавлять карточку в публичный sitemap;
- публиковать рабочие характеристики как подтверждённые;
- обещать цену, наличие, срок, ипотеку или юридическую схему;
- использовать изображения без подтверждённых прав.

## Канонический формат

Verification-профили хранятся только в:

```text
data/verification/prostornaya-4a.json
data/verification/aerodromnaya-18g.json
data/verification/sennaya-76.json
```

Контракт:

```text
data/verification/profile-contract.json
```

Каждый claim обязан содержать:

```text
field
value
importance
verification_status
source_ids
checked_at
publication_allowed
```

Рабочие, вторичные, пользовательские, противоречивые и не найденные сведения имеют `publication_allowed=false`.


Сводка проверки:

```text
Источники: 0 из 3 проверены

```

Рабочий реестр содержит номера карточки ЕИСЖС, проектной декларации и разрешения на строительство. Точные публичные ссылки пока не приняты, поэтому все значения остаются `working_copy` и не разрешены к публикации.

Следующий минимальный набор:

1. Сохранить точную ссылку на карточку ЕИСЖС / наш.дом.рф.
2. Сохранить актуальную редакцию проектной декларации.
3. Сохранить официальный файл или ссылку разрешения на строительство.
4. Повторно проверить 14 critical claims.
5. Отдельно подтвердить права на визуализации.

## Источники и очередь проверки

Наличие verification-профиля не означает завершение сбора документов.

Отдельная очередь сохраняет:

```text
projects=3
tasks=14
accepted=0
missing=14
```

Источник считается принятым только после сохранения точной HTTPS-ссылки или официального файла, даты проверки и доказательной заметки.

## Что разрешено сейчас

Для всех трёх объектов разрешено:

- собирать предварительные заявки;
- предлагать проверку документов и условий покупки;
- показывать дату последней внутренней актуализации;
- объяснять границу между рабочей и подтверждённой информацией;
- предлагать общий подбор и предварительный финансовый расчёт.

## Что запрещено считать подтверждённым

Без принятого первичного источника нельзя публиковать как факт:

- цену и наличие;
- этажность и квартирографию;
- ввод объекта или отдельной секции;
- срок передачи;
- застройщика или продавца;
- тип договора;
- возможность ипотеки;
- скидки, бронь и очередь;
- официальные планировки;
- права на изображения.

## Команды

Проверить все verification-профили:

```bash
npm run validate:object-verification
```

Сформировать readiness-отчёт:

```bash
npm run projects:readiness
```

JSON:

```bash
npm run projects:readiness -- --format=json
```

CSV:

```bash
npm run projects:readiness -- --format=csv
```

Проверить согласованность полного контура:

```bash
npm run validate:verification
```

## Правило обновления

После принятия источника или изменения claim необходимо одновременно обновить:

1. verification-профиль;
2. проектный профиль и индекс, если меняется публичная готовность;
3. очередь источников;
4. этот отчёт;
5. дату проверки;
6. публичную карточку только в пределах `publication_allowed=true`.
