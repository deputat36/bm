### Источники без сохранённой ссылки

1. Карточка объекта в ЕИСЖС / наш.дом.рф.
2. Актуальная проектная декларация.
3. Разрешение на строительство.

В реестре сохранены номера документов и рабочие значения, но поля `reference` у всех трёх источников пустые.

### Critical claims, которые нельзя считать подтверждёнными

- адрес;
- юридическое лицо застройщика;
- статус строительства;
- этажность;
- общее количество квартир;
- минимальная и максимальная площадь;
- срок ввода;
- срок передачи ключей;
- номер и дата проектной декларации;
- ID объекта на наш.дом.рф;
- номер и дата разрешения на строительство.

### Следующий минимальный набор действий

1. Сохранить точную публичную ссылку на карточку объекта.
2. Сохранить ссылку на актуальную редакцию проектной декларации.
3. Сохранить ссылку или файл разрешения на строительство.
4. Повторно сверить 14 critical claims.
5. Только после этого менять отдельные claims на `confirmed`.

Пока хотя бы один critical claim не подтверждён, страница остаётся `noindex,follow`, а `is_public_ready` остаётся `false`.


Пока подтверждены только рабочий адрес и факт существования карточки портала. Застройщик, документы, статус, сроки, характеристики и квартирография не подтверждены первичными источниками.

Нужно собрать пять категорий источников:

1. Карточка объекта или дома в публичном источнике.
2. Документы застройщика, если доступны.
3. Разрешение на строительство, если объект строится.
4. Проектная декларация, если продажи идут по ДДУ.
5. Фотографии и материалы с подтверждёнными правами на публикацию.

После появления источников необходимо создать отдельный verification-файл и описать claims по той же модели, что используется для Просторной 4А.

## Что разрешено сейчас

Для всех трёх объектов разрешено:

- показывать безопасный статус «данные уточняются»;
- собирать предварительные заявки и вопросы;
- объяснять, какие сведения будут проверены специалистом;
- вести внутренний реестр источников;
- использовать страницы в закрытом `noindex,follow` контуре.

## Что запрещено считать подтверждённым

Без первичного источника нельзя публиковать как факт:

- текущую цену;
- наличие конкретных квартир;
- скидки;
- бронь или очередь;
- точные сроки старта продаж;
- окончательный внешний вид;
- официальные планировки;
- ипотечное одобрение;
- гарантированные места или инфраструктуру.

## Команды

Сформировать отчёт:

```bash
npm run projects:readiness
```

JSON:

```bash
npm run projects:readiness -- --format=json
```

CSV:

```bash
npm run projects:readiness -- --format=csv
```

Только объекты, которым нужны источники:

```bash
npm run projects:readiness -- --status=requires_sources
```

Проверить согласованность отчёта:

```bash
npm run validate:project-readiness
```

## Правило обновления

После добавления ссылки, verification-файла или подтверждения claim необходимо одновременно обновить:

1. проектный профиль;
2. verification-файл;
3. этот отчёт;
4. дату проверки;
5. публичную карточку только в пределах подтверждённых сведений.
