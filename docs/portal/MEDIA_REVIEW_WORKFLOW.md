# Проверка визуальных материалов портала

Дата обновления: 2026-07-13

## Цель

Визуальные материалы нельзя автоматически переносить из старого сайта в новый портал. Для каждого изображения нужно подтвердить происхождение, актуальность, связь с конкретным объектом и право публикации.

Машиночитаемый реестр Просторной 4А:

```text
data/media/prostornaya-4a.json
```

Автоматическая проверка:

```bash
npm run validate:media
```

## Текущий статус

В репозитории обнаружено семь архитектурных визуализаций с именами `Borisoglebsk_001`–`Borisoglebsk_007`.

Для всех материалов установлен безопасный статус:

```text
verification_status: requires_check
rights_status: unknown
is_public_ready: false
allowed_usage: []
```

До завершения проверки они не должны автоматически появляться на главной, в каталоге, рекламе, социальных сетях или карточке объекта.

## Условия публикации

Изображение можно перевести в публичный статус только после выполнения всех условий:

1. Установлен автор, правообладатель или подтверждённый источник.
2. Есть основание для публикации на портале.
3. Подтверждено, что материал относится к актуальной версии нужного объекта.
4. Проверено, что изображение не смешивает несколько корпусов или этапов без пояснения.
5. Визуализация не выдаётся за фотографию построенного объекта.
6. Описание не обещает фактический вид фасада, двора, парковки или благоустройства.
7. Заполнены `source_reference`, `verification_status`, `rights_status` и `allowed_usage`.
8. `is_public_ready` явно установлен в `true`.

Минимальный публичный набор полей:

```json
{
  "verification_status": "confirmed",
  "rights_status": "cleared",
  "source_reference": "ссылка или номер предоставленного материала",
  "is_public_ready": true,
  "allowed_usage": ["project_page"]
}
```

## Допустимые сценарии использования

```text
catalog
project_page
news
guides
social
advertising
```

Разрешение на один сценарий не означает автоматического разрешения на остальные. Например, материал может быть допустим в карточке объекта, но не согласован для платной рекламы.

## Работа галереи

`assets/js/gallery.js` больше не содержит жёстко заданного списка файлов. Скрипт загружает реестр и выводит только материалы, одновременно соответствующие условиям:

```text
verification_status=confirmed
rights_status=cleared
is_public_ready=true
source_reference заполнен
allowed_usage не пуст
```

Если подходящих материалов нет, пользователь получает нейтральное сообщение о проверке галереи, а не неподтверждённые рендеры.

## Что проверяет валидатор

`tools/validate-media-inventory.mjs` контролирует:

- существование основных и резервных файлов;
- уникальность ID и путей;
- допустимые типы и статусы;
- заполненность заголовка и alt-текста;
- наличие источника и прав у публичных материалов;
- отсутствие обхода реестра через жёстко заданные имена файлов;
- отсутствие подключения галереи на активных страницах, если публично готовых материалов нет.

## Следующий шаг

Нужно запросить у владельца материалов или застройщика подтверждение:

```text
кто создал изображения;
когда они подготовлены;
к какой версии проекта относятся;
на каких площадках разрешено использование;
можно ли изменять, кадрировать и снабжать текстом;
есть ли более свежая редакция.
```

После получения подтверждения каждое изображение проверяется отдельно. Массовое открытие всех файлов одним действием запрещено.
