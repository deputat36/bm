# Verification-безопасность главной страницы

Дата обновления: 2026-07-15

## Причина изменения

Корневая страница портала индексируется, а страницы отдельных объектов сохраняют `noindex,follow` до прохождения проверки источников.

Отчёт готовности фиксирует:

```text
public_ready=0/3
Просторная 4А: confirmed critical claims=0/14
```

При таком статусе индексируемая главная не должна публиковать точные характеристики проекта как подтверждённые факты.

## Удалённые утверждения

Из карточки Просторной 4А удалены:

- статус строительства;
- связь с конкретным жилым комплексом как установленный факт;
- этажность;
- количество квартир;
- квартирография;
- диапазон площадей;
- квартал сдачи;
- дата передачи ключей.

Эти сведения могут вернуться только после сохранения первичных ссылок и подтверждения соответствующих claims в verification-файле.

## Разрешённая карточка

Пока источники не приняты, карточка может:

- использовать рабочее название объекта;
- показывать статус «Данные уточняются»;
- объяснять, какие категории сведений проверяются;
- собирать предварительные обращения;
- направлять на закрытую `noindex,follow` страницу объекта;
- сообщать, что актуальные подтверждённые данные будут озвучены на момент обращения.

## Автоматическая проверка

Файл:

```text
tools/validate-homepage-verification-safety.mjs
```

Guard сверяет индексируемую главную с:

```text
docs/portal/PROJECT_VERIFICATION_READINESS.md
```

Пока отчёт содержит `0/3 public_ready` и `0/14 confirmed critical claims` для Просторной 4А, проверка требует:

- статус «Данные уточняются» у трёх карточек;
- безопасный текст карточки Просторной 4А;
- отсутствие числовых характеристик объекта;
- отсутствие дат сдачи и передачи;
- отсутствие старых неподтверждённых формулировок.

Если хотя бы один объект станет `public_ready` или появятся подтверждённые critical claims, guard намеренно остановит CI. После этого требуется вручную обновить разрешённый набор публичных утверждений.

## Ограничения

Guard не подтверждает сведения об объекте и не заменяет первичные документы. Он только не позволяет индексируемой странице опережать текущий verification-статус.
