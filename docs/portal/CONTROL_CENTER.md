## 1. Цель проекта

Превратить текущий проект из сайта одного ЖК в городской портал:

```text
novostroyki-borisoglebsk.ru
```

Главная страница будущего портала должна быть не лендингом одного объекта, а каталогом всех новостроек Борисоглебска.

Портал должен выглядеть как независимый городской каталог:

- список объектов;
- карточки домов и ЖК;
- сравнение адресов, сроков, документов и способов покупки;
- справочник покупателя;
- новости по новостройкам;
- формы заявок на подбор квартиры и консультацию.

## 5. Правила миграции

1. Сначала создать новую структуру как черновики с `noindex,follow`.
2. Не менять корневой `index.html` до готовности каталога.
3. Не менять `sitemap.xml` до финального решения по домену и URL.
4. Не менять canonical до утверждения структуры.
5. Не удалять старые URL.
6. Старые URL переводить через карту редиректов только после проверки новых страниц.
7. Все новые страницы должны быть юридически безопасными и не выглядеть как официальный сайт застройщика.
8. Главная будущего портала должна быть каталогом всех новостроек, а не страницей одного ЖК.

Текущий объект, связанный с ЖК «Теллерманов сад».

Статус: данные частично подготовлены, но нужно привести URL и позиционирование к формату каталога.

### Аэродромная 18Г

Статус: карточка должна быть создана как отдельный объект каталога.

Требуется заполнение данных:

- адрес;

- статус;
- сроки;
- квартирография;
- документы;
- фото / рендеры;
- дисклеймеры.

### Сенная 76

Статус: карточка должна быть создана как отдельный объект каталога.

Требуется заполнение данных по той же модели, что и для других объектов.

## 8. Ближайший план работ

1. Создать `docs/portal/ARCHITECTURE.md` с целевой архитектурой.
2. Подготовить карту миграции старых URL в новую структуру `/catalog/`.
3. Создать черновой `/catalog/index.html` с `noindex,follow`.
4. Создать черновые карточки:
   - `/catalog/prostornaya-4a/`
   - `/catalog/aerodromnaya-18g/`
   - `/catalog/sennaya-76/`
5. Создать алиасы или будущие правила миграции:
   - `/novostroyki/` → `/catalog/`
   - `/zhk/tellermanov-sad/` → `/catalog/prostornaya-4a/`
   - `/zastroyschiki/` → `/developers/`
   - `/spravochnik/` → `/guides/`
   - `/novosti/` → `/news/`
6. После создания черновиков запустить проверки:

```bash
npm run validate
npm run validate:json
```
