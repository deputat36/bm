# Панель ручного QA лид-форм

## Назначение

Внутренний инструмент:

```text
/tools/form-qa-runner.html
```

помогает выполнить матрицу из 14 сценариев на трёх устройствах:

- desktop;
- Android;
- iPhone.

Всего требуется проверить 42 слота. Панель не является доказательством прохождения тестов сама по себе.

## Безопасный режим

Каждая кнопка «Открыть dry-run» формирует ссылку с обязательными параметрами:

```text
lead_test=dry-run
analytics_test=debug
test_ack=1
```

При корректной работе страницы сверху отображается красный баннер тестового режима. Внешняя доставка заявки запрещена контрактом.

## Что хранится

Прогресс сохраняется только в `localStorage` текущего браузера под ключом:

```text
newbuildsBorisoglebskFormQaRunnerV1
```

Хранятся только:

- выбранное устройство;
- браузер и версия;
- ОС и версия;
- отметки чек-листа;
- итог проверки;
- ссылка или идентификатор обезличенного evidence;
- отметка о приложенном event log;
- техническая заметка.

Имя, телефон, email, содержание тестовой заявки и `user_agent` панель не собирает.

## Требования к каждому слоту

Контракт `data/qa/form-execution-contract.json` содержит десять обязательных проверок:

1. открыта правильная форма;
2. работает required-валидация и фокус;
3. проверяются границы длины телефона;
4. корректны мобильная клавиатура и autocomplete;
5. доступны select и текстовые поля;
6. dry-run завершается без реальной доставки;
7. сформирована обязательная последовательность аналитических событий;
8. event log не содержит персональных или лишних технических данных;
9. повторная отправка защищена;
10. status-region и кнопка восстанавливаются.

Статус `passed` допустим только когда:

- отмечены все десять проверок;
- указан браузер;
- указана ОС;
- заполнен `evidence_reference`;
- включён `event_log_attached`.

Для `failed` и `blocked` обязательна заметка.

## Проверки устойчивости

Отдельно один раз на каждом устройстве проверяются:

- недоступный `localStorage`;
- недоступный `sessionStorage`.

Они входят в evidence export, но не заменяют 42 сценария форм.

## Экспорты

### form-results.json

Кнопка формирует файл, совместимый с:

```text
data/qa/form-results.json
```

Локальные фактически заполненные записи объединяются с уже существующими записями репозитория по ключу:

```text
scenario_id + device
```

Отсутствующая запись означает `not_run`.

### form-qa-evidence.json

Содержит:

- среду каждого устройства;
- результаты чек-листов;
- ссылки на evidence;
- технические заметки;
- результаты storage-проверок.

Статус файла:

```text
local_evidence_export_no_implied_approval
```

Экспорт не означает одобрение и не меняет manual gates.

## Запрещено

- вводить реальные персональные данные;
- прикладывать скриншоты с телефоном, email или именем;
- использовать панель для реальной отправки;
- считать локальные отметки результатом без evidence;
- вручную менять `not_run` на `passed` без фактической проверки;
- закрывать issue #71 только на основании созданного инструмента.

## Автоматическая защита

Workflow:

```text
.github/workflows/form-qa-runner.yml
```

проверяет:

- `noindex,nofollow,noarchive`;
- отсутствие лид-форм и полей персональных данных;
- отсутствие Web3Forms, POST и `sendBeacon`;
- обязательный dry-run;
- 14 сценариев, 3 устройства и 10 проверок;
- 42 итоговых слота;
- сохранение запрета внешней доставки;
- валидность текущего `form-results.json`.
