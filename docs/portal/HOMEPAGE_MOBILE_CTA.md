# Мобильная панель конверсии главной страницы

## Цель

Сократить путь посетителя с мобильного поиска до звонка или короткой заявки без создания новых форм и без изменения неподтверждённых сведений об объектах.

## Реализация

На ширине до `760px` используются два уже существующих элемента главной страницы:

- телефонная ссылка первого экрана;
- кнопка перехода к форме `#quick-lead` из основного меню.

CSS фиксирует их в нижней части экрана и добавляет фоновую панель. Отдельный HTML-блок и отдельная форма не создаются.

## Ограничение области

Панель включается только на странице, где присутствует:

```text
.hero__content--conversion
```

Для этого используется селектор:

```css
body:has(.hero__content--conversion)
```

На остальных страницах `home-polish.css` не создаёт нижнюю панель.

## Доступность и удобство

- учтена безопасная нижняя зона iPhone через `env(safe-area-inset-bottom)`;
- фон панели не перехватывает нажатия;
- кнопки занимают по половине доступной ширины;
- якоря форм имеют дополнительный `scroll-margin-bottom`, чтобы панель не перекрывала начало формы;
- при отсутствии поддержки `:has()` страница остаётся полностью работоспособной без закрепления.

## Аналитика

Сохраняются действующие события и размещения:

```text
phone / hero
quick_selection / header
```

Новые события, формы, query-параметры и хранилища не добавляются.
