# Агрегированный статус проверки объектов

Дата обновления: 2026-07-16

## Назначение

Карточки Просторной 4А, Аэродромной 18Г и Сенной 76 показывают посетителю границу достоверности данных без вывода внутренних claim values.

Блок отвечает только на три вопроса:

1. Когда внутренний профиль обновлялся.
2. Сколько источников принято как проверенные.
3. Сколько critical claims подтверждено.

## Публичный формат

Пример:

```text
Внутренняя актуализация: 16.07.2026.
Проверено источников: 0 из 6.
Подтверждено критических фактов: 0 из 8.
```

Дополнительно выводится предупреждение:

```text
Точные характеристики не публикуются до принятия первичных источников и завершения юридической проверки.
```

## Источник данных

Каждая страница содержит ссылку на свой профиль:

```text
data-verification-profile="../../data/verification/<slug>.json"
```

Runtime:

```text
assets/js/project-verification-summary.js
```

## Что нельзя выводить

Runtime не выводит:

- `claim.value`;
- названия и заметки источников;
- прямые ссылки на документы;
- рабочие цены и наличие;
- сведения пользователя;
- противоречия и legal notes;
- внутренние идентификаторы заявки.

## Защита

Validator:

```text
tools/validate-project-verification-summary.mjs
```

Он проверяет:

- три страницы и точные пути к профилям;
- единственное подключение runtime;
- порядок `main.js → summary runtime → schema.js`;
- отсутствие `innerHTML` и вывода деталей профиля;
- текущие агрегаты profiles;
- `publication_allowed=false` у всех claims;
- сохранение `noindex,follow`.

При появлении первого verified source или confirmed critical claim CI намеренно остановится. Это требует осознанно обновить отображаемую границу достоверности и readiness-документацию.

## Ошибка загрузки

Если JSON-профиль недоступен, страница не показывает нулевые показатели как достоверные. Вместо этого выводится:

```text
Статус проверки временно недоступен. Уточните актуальные сведения перед консультацией.
```

## Команда

```bash
npm run validate:project-verification-summary
```
