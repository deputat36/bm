# Сохранение объекта в ипотечной воронке

Дата обновления: 2026-07-14

## Проблема

На страницах Просторной 4А, Аэродромной 18Г и Сенной 76 кнопки ипотечного расчёта изначально вели на `/ipoteka/` без контекста объекта.

После появления короткой ипотечной формы возник второй риск: старый JavaScript-якорь `#lead` мог перескакивать через первичный сценарий сразу к подробному расчёту.

## Текущий сценарий

```text
страница объекта
→ CTA «Ипотечный расчёт»
→ /ipoteka/?object=<residential_complex_id>#quick-lead
→ объект автоматически выбран в короткой и подробной формах
→ residential_complex_id синхронизирован
→ пользователь видит подтверждение выбранного объекта
→ короткая ипотечная консультация
→ уточнение программы и финансовых параметров
```

Подробная форма остаётся доступной ниже на странице, но объектный CTA не направляет к ней автоматически.

## Формирование ссылки

`assets/js/conversion-tracking.js` обрабатывает ссылки:

```text
data-track-action="mortgage_open"
data-track-object="prostornaya-4a | aerodromnaya-18g | sennaya-76"
```

Перед переходом добавляется:

```text
?object=<data-track-object>#quick-lead
```

Основной якорь задан явно:

```text
MORTGAGE_PRIMARY_ANCHOR=quick-lead
```

Старый вариант:

```text
?object=<data-track-object>#lead
```

запрещён автоматическим guard, поскольку `#lead` относится к подробной форме.

Исходный относительный путь сохраняется, поэтому механизм работает в локальной структуре сайта и на основном домене.

## Автоподстановка

`assets/js/priority-leads.js`:

1. Читает `object`, `project` или `complex` из query string.
2. Сопоставляет значение с безопасным реестром приоритетных объектов.
3. Выбирает объект во всех формах с полем `residential_complex`.
4. Записывает соответствующий `residential_complex_id`.
5. Обновляет `data-complex` и `data-complex-id` формы.
6. Показывает сообщение `data-selected-object-note`.

Параметр не выводится как произвольный пользовательский текст. На странице показывается только название из внутреннего списка допустимых объектов.

## Формы

Первичная форма:

```text
form_id=portal_mortgage_quick_consultation
lead_type=mortgage
```

Она запрашивает:

```text
имя
телефон
объект
```

Подробная форма:

```text
form_id=portal_mortgage_consultation
lead_type=mortgage
```

Она дополнительно собирает программу, комнатность, бюджет или первоначальный взнос, срок и комментарий.

## Сообщение покупателю

На ипотечной странице отображается уведомление:

```text
Вы перешли к предварительному расчёту по выбранному объекту. Он уже выбран в форме. Цена, наличие и возможность применения программы уточняются отдельно на дату консультации.
```

Сообщение:

- не обещает одобрение ипотеки;
- не подтверждает цену или наличие;
- не создаёт впечатление бронирования;
- объясняет, почему итоговые условия нужно уточнять.

## Аналитика

Событие `lead_cta_click` получает уже обогащённый `link_url`, а также:

```text
action=mortgage_open
placement=project_hero | project_faq
object_id=<residential_complex_id>
```

После перехода первичной формой является:

```text
portal_mortgage_quick_consultation
```

Это позволяет сравнивать:

```text
lead_cta_click
→ lead_form_view
→ lead_form_start
→ lead_submit
```

без смешивания первичной консультации и подробного расчёта.

## Автоматическая проверка

`npm run validate:mortgage-conversion` проверяет:

- наличие короткой и подробной ипотечной формы;
- приоритет `data-primary-lead`;
- якорь `#quick-lead` для объектных ипотечных CTA;
- запрет старого якоря `#lead` в механизме обогащения ссылок;
- установку query-параметра `object`;
- автоподстановку объекта в обе формы;
- синхронизацию `residential_complex_id`;
- наличие безопасного уведомления;
- обязательность программы только в подробной форме.

`validate:utm` дополнительно требует, чтобы активная ипотечная рекламная кампания была привязана к:

```text
portal_mortgage_quick_consultation
```

## Ограничения

Механизм не:

- проверяет фактическую доступность ипотечной программы;
- подтверждает аккредитацию объекта банком;
- рассчитывает ставку;
- отправляет заявку в банк;
- гарантирует одобрение.

Эти вопросы решаются отдельно после уточнения программы, банка, параметров покупателя и актуального статуса объекта.
