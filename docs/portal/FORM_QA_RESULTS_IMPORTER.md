# Импорт результатов ручного QA

## Назначение

`tools/import-form-qa-results.mjs` переносит фактически выполненные проверки из экспортированного панелью файла в `data/qa/form-results.json`.

Инструмент не проводит тесты и не выставляет результаты самостоятельно. Он только проверяет и объединяет уже зафиксированные обезличенные данные.

## Исходные файлы

Панель `/tools/form-qa-runner.html` формирует два файла:

- `form-results.json` — записи для реестра;
- `form-qa-evidence.json` — обезличенный evidence-пакет с десятью проверками каждого слота.

Для статуса `passed` необходимо передать оба файла. `failed` и `blocked` можно импортировать без evidence-пакета, но для них обязательна содержательная заметка.

## Предпросмотр

По умолчанию реестр не изменяется:

```bash
node tools/import-form-qa-results.mjs \
  --input=/путь/form-results.json \
  --evidence=/путь/form-qa-evidence.json
```

Результат команды показывает:

- сколько записей уже есть;
- сколько записей обнаружено во входном файле;
- сколько слотов будет добавлено;
- сколько совпадает без изменений;
- сколько требует замены;
- итоговое заполнение из 42 слотов.

Для сохранения отдельного файла предпросмотра:

```bash
node tools/import-form-qa-results.mjs \
  --input=/путь/form-results.json \
  --evidence=/путь/form-qa-evidence.json \
  --output=/путь/form-results-preview.json
```

## Запись в реестр

После проверки предпросмотра:

```bash
node tools/import-form-qa-results.mjs \
  --input=/путь/form-results.json \
  --evidence=/путь/form-qa-evidence.json \
  --write
```

Команда обновит только `data/qa/form-results.json`.

После записи обязательно выполнить:

```bash
node tools/validate-form-qa-results.mjs
node tools/build-form-qa-report.mjs --format=json
```

## Замена существующего результата

Отличающийся существующий слот не заменяется автоматически.

Для осознанной повторной проверки используется:

```bash
node tools/import-form-qa-results.mjs \
  --input=/путь/form-results.json \
  --evidence=/путь/form-qa-evidence.json \
  --replace-existing \
  --write
```

Замена разрешена только при более новом `tested_at`. Это защищает реестр от случайного возврата к старому экспорту.

## Проверки безопасности

Импорт блокируется, если:

- неизвестен сценарий или устройство;
- используется статус, отличный от `passed`, `failed`, `blocked`;
- повторяется пара `scenario_id + device`;
- дата не является точным ISO datetime;
- для `passed` отсутствует evidence reference;
- для `passed` не приложен event log;
- для `passed` отсутствует соответствующая запись evidence-пакета;
- хотя бы одна из десяти обязательных проверок не подтверждена;
- данные реестра и evidence-пакета различаются;
- для `failed` или `blocked` отсутствует заметка;
- обнаружено имя, телефон, email, user agent, URL, UTM или другое запрещённое техническое поле;
- входной файл содержит неизвестные поля записи;
- конфликтующий результат заменяется без `--replace-existing`;
- новая запись замены имеет не более позднее время проверки;
- итоговый реестр превышает 42 слота.

## Самотест

Для проверки импортёра используется отдельный пустой fixture, а не рабочий реестр:

```bash
node tools/import-form-qa-results.mjs \
  --self-test \
  --registry=tools/fixtures/form-results-empty.json
```

Самотест проверяет:

- успешное добавление корректного `passed`;
- повторный импорт без дубля;
- блокировку `passed` без evidence;
- блокировку персонального поля;
- блокировку тихой замены;
- разрешение явной более новой замены;
- отсутствие записи без `--write`.

## Ограничения

Импортёр не подтверждает:

- фактическую работу страницы в браузере;
- реальную доставку заявки;
- работу GA4 или Яндекс Метрики;
- достоверность изображения или записи экрана;
- решение владельца о прохождении manual gate.

Статусы P0 и launch-readiness обновляются только после фактического выполнения всех требуемых проверок и отдельного просмотра evidence.
