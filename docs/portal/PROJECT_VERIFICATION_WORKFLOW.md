# Проверка данных объекта перед публикацией

Дата обновления: 2026-07-13

## Назначение

Для Просторной 4А недостаточно общего статуса «частично подтверждено». Каждая существенная характеристика должна быть связана с конкретным первичным источником и иметь собственный статус проверки.

Рабочий реестр:

```text
data/verification/prostornaya-4a.json
```

Автоматическая проверка:

```bash
npm run validate:verification
```

## Источники

В реестре предусмотрены три базовых источника:

```text
карточка объекта ЕИСЖС / наш.дом.рф
актуальная проектная декларация
разрешение на строительство
```

Для каждого источника нужно сохранить точную публичную ссылку или иной проверяемый идентификатор. Номер документа без ссылки не считается завершённой проверкой.

Допустимые статусы источника:

```text
reference_required
attached_unverified
verified
rejected
```

## Проверяемые характеристики

Реестр охватывает:

```text
адрес
застройщика
статус строительства
статус продаж
класс
материал стен
этажность
подъезды
общее число квартир
минимальную и максимальную площадь
высоту потолков
срок ввода
срок передачи ключей
номер и дату декларации
ID объекта ЕИСЖС
номер и дату разрешения
квартирографию по типам квартир
```

Допустимые статусы характеристики:

```text
working_copy
confirmed
rejected
not_applicable
```

`working_copy` означает, что значение может использоваться только на закрытой от индексации рабочей странице с предупреждением о повторной проверке.

## Условия подтверждения

Характеристика переводится в `confirmed` только когда:

1. значение совпадает с актуальным первичным источником;
2. все связанные источники имеют статус `verified`;
3. у источников заполнено поле `reference`;
4. проверка выполнена для актуальной редакции документа;
5. отсутствуют противоречия между источниками.

## Открытие страницы для индексации

Страница Просторной 4А остаётся `noindex,follow`, пока не подтверждены все характеристики с важностью `critical`.

Даже после этого отдельно проверяются:

```text
цены
наличие квартир
скидки
бронирование
условия ипотеки
визуальные материалы
```

Эти сведения не входят в документную проверку и требуют актуализации непосредственно перед публикацией или рекламным запуском.

## Что проверяет валидатор

`tools/validate-project-verification.mjs` контролирует:

- связь проектного профиля с реестром проверки;
- совпадение значений в профиле и claims;
- наличие обязательных характеристик;
- существование всех указанных источников;
- невозможность подтвердить claim без проверенного источника и ссылки;
- сохранение `is_public_ready=false` и `noindex,follow`, пока critical claims не подтверждены;
- явное исключение цен, наличия, бронирования и других неподтверждённых обещаний.

## Следующий практический шаг

Нужно получить и сохранить точные ссылки на:

```text
актуальную карточку объекта ЕИСЖС;
актуальную редакцию проектной декларации №36-001139;
разрешение на строительство №36-04-13-2026.
```

После этого значения сверяются по одному. Массовая смена всех статусов на `confirmed` без построчной проверки запрещена.
