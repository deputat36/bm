# CTA объектов на главной странице

Дата: 2026-07-14

## Проблема

После внедрения коротких форм на страницах трёх приоритетных объектов карточки главной страницы продолжали вести к подробным формам `#lead`.

Из-за этого пользователь, выбравший конкретный объект на главной, обходил более короткий сценарий первого обращения.

Старый путь:

```text
главная → карточка объекта → подробная форма
```

## Реализованный путь

Основные кнопки карточек направлены к коротким формам:

```text
главная → карточка объекта → #quick-lead → имя + телефон + главный вопрос
```

Целевые страницы:

```text
/catalog/prostornaya-4a/#quick-lead
/catalog/aerodromnaya-18g/#quick-lead
/catalog/sennaya-76/#quick-lead
```

Подробные формы на страницах объектов сохранены ниже и доступны посетителям, которые готовы указать бюджет, способ и срок покупки.

## Аналитика

Основные CTA карточек используют:

```text
action=object_quick_consultation
placement=homepage_object_card
object_id=<residential_complex_id>
```

Ссылки «Подробнее» используют:

```text
action=object_details
placement=homepage_object_card
object_id=<residential_complex_id>
```

Ссылка на общий каталог после карточек использует:

```text
action=catalog_open
placement=homepage_objects
```

Персональные данные в события не передаются.

## Автоматическая защита

Файл:

```text
tools/validate-homepage-object-cta.mjs
```

Проверяет:

- наличие короткой формы `homepage_quick_selection`;
- переход каждой карточки к `#quick-lead`;
- отсутствие старого перехода к `#lead`;
- корректный `residential_complex_id` в аналитической разметке;
- разметку ссылок «Подробнее»;
- наличие `id="quick-lead"` и `data-primary-lead` на целевой странице;
- отсутствие устаревшего события `object_lead`;
- разметку ссылки на общий каталог.

Команда:

```bash
npm run validate:homepage-cta
```

Проверка также включена в общий `npm run validate` и отдельным шагом в Portal guards.

## Дополнительное усиление CI

В Portal guards отдельными шагами теперь запускаются:

```text
validate:homepage-cta
validate:catalog-conversion
validate:thankyou
```

Это исключает ситуацию, когда специализированный guard существует, но не выполняется в основном workflow портала.

## Ограничения проверки

Автоматические проверки подтверждают структуру HTML, маршруты и аналитические атрибуты. Они не заменяют ручную проверку клика, прокрутки и отображения формы на физических Android и iPhone.
