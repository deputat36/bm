# Контекст страницы благодарности

Дата обновления: 2026-07-15

## Правила

Техническая запись `newbuildsBorisoglebskLastLead` используется для сопоставления объекта и формы на `/spasibo/`.

Запись считается действительной не более 24 часов. Допускается отклонение часов браузера до пяти минут в будущее. Некорректная, повреждённая или устаревшая запись удаляется.

Допустимые ID:

```text
NB-YYYYMMDD-XXXXXX
NB-TEST-YYYYMMDD-XXXXXX
```

Произвольный параметр `id` удаляется из URL до запуска основной логики страницы.

## Dry-run

`dry_run=1` принимается только когда:

- ID имеет формат `NB-TEST-...`;
- существует свежая локальная тестовая запись;
- ID в URL совпадает с ID записи;
- запись содержит `dry_run=true`.

Иначе удаляются `dry_run`, `lead_test`, `analytics_test` и `test_ack`. Поэтому вручную добавленный тестовый параметр не может подавить обычное событие страницы благодарности.

## Порядок

Hardening выполняется в `assets/js/analytics-debug.js` до чтения параметров analytics debug:

```text
проверка пути
→ проверка lastLead
→ очистка URL
→ определение debug
→ основная логика страницы
```

На остальных страницах механизм не активируется.

## Проверка

`tools/validate-thankyou-context.mjs` контролирует:

- 24-часовой срок;
- допустимые форматы ID;
- удаление повреждённой записи;
- очистку URL;
- сопоставление тестового ID;
- порядок выполнения;
- отсутствие технического ID в аналитике благодарности.

Ручной QA должен отдельно проверить свежую, устаревшую и повреждённую запись, произвольный ID, тестовый URL без локального контекста и недоступный `localStorage`.
