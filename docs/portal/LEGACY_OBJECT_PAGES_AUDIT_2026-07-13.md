# Аудит legacy-страниц объекта Просторная 4А

Дата: 2026-07-13

Репозиторий: `deputat36/bm`

## Цель

Старый сайт содержал несколько конкурирующих страниц одного объекта. Новый портал использует единую карточку:

```text
/catalog/prostornaya-4a/
```

Задача аудита — убрать дубли контента, старый брендинг и устаревшие формы, не включая преждевременные серверные редиректы.

## Критические исправления

`/prostornaya-4a/` ранее содержал canonical домена `tellermanovsad.ru` и не имел `noindex`. Теперь он закрыт от индексации и ведёт на портальную карточку.

`/zhk/tellermanov-sad/` ранее дублировал карточку объекта, содержал старую навигацию и отдельную форму `tellermanov_sad_portal_page`. Дублирующая форма удалена, чтобы все обращения по объекту проходили через `catalog_prostornaya_4a_priority_lead`.

## Почему страницы объединяются

Старые URL дробили один объект на несколько тонких страниц и создавали риски:

1. дублирование характеристик;
2. расхождение цен, площадей и сроков;
3. конкуренция страниц в поиске;
4. сохранение брендинга старого проекта;
5. заявки без единого `form_id` и системного ID объекта;
6. публикация устаревших сведений после изменения источника.

Единая карточка позволяет хранить статус проверки, форму обращения и юридические ограничения в одном месте.

## Статус будущего редиректа

Все семь маршрутов зарегистрированы в:

```text
data/migration/legacy-routes.json
```

Текущий статус:

```text
status: transition_page
redirect_phase: 2
redirect_ready: false
```

Редирект нельзя выпускать до проверки:

1. актуальной карточки ЕИСЖС и проектной декларации;
2. формы `catalog_prostornaya_4a_priority_lead`;
3. мобильного отображения;
4. способа настройки серверных правил;
5. сохранения UTM-параметров;
6. плана отката.

## Остались на аудит

```text
/galereya/
/infrastruktura/
/hod-stroitelstva/
/zastroyschik/
/proektnaya-deklaratsiya-36-001139/
/faq/
```

Предварительное решение:

- `galereya`, `infrastruktura`, `hod-stroitelstva` — оценить самостоятельную информационную ценность;
- `zastroyschik` — переносить только в подтверждённую карточку застройщика;
- документную страницу — сохранить либо объединить после повторной проверки первичного источника;
- `faq` — разделить общие вопросы и вопросы конкретного объекта.

## Автоматический контроль

Команда:

```bash
npm run validate:legacy
```

Проверяет существование файлов, `noindex`, ссылку на целевой URL, отсутствие старого домена, брендинга, устаревших лид-форм и автоматических перенаправлений.
