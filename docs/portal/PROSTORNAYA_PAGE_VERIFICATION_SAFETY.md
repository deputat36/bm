# Verification-безопасность страницы Просторной 4А

Дата обновления: 2026-07-16

## Причина

Verification-профиль Просторной 4А содержит 23 рабочих claim, включая этажность, квартирографию, площади, сроки и реквизиты документов.

Пока:

```text
publication_allowed=0/23
confirmed critical claims=0/14
```

эти сведения не должны выводиться на пользовательской странице даже с оговоркой «рабочие данные».

## Что удалено из публичного HTML

- связь с названием ЖК как установленный факт;
- этажность и количество подъездов;
- материал стен и лифт;
- количество и типы квартир;
- диапазон площадей;
- срок сдачи и передачи ключей;
- номер и дата проектной декларации;
- ID карточки ЕИСЖС;
- номер и дата разрешения на строительство;
- `data-schema-floors`.

## Что осталось

Страница сохраняет:

- адрес как идентификатор обращения;
- безопасный статус «Данные уточняются»;
- описание категорий, которые проверяются;
- список недостающих первичных источников;
- короткую и подробную формы;
- телефонный CTA;
- предварительный ипотечный расчёт;
- `noindex,follow`.

## Guard

Файл:

```text
tools/validate-prostornaya-page-verification-safety.mjs
```

Проверка:

- читает `data/verification/prostornaya-4a.json`;
- требует 23 claims и 0 публичных claims;
- требует 0 подтверждённых critical claims;
- запрещает старые числа, даты и номера документов в HTML;
- проверяет безопасные тексты;
- подтверждает сохранение двух форм и объектной аналитической разметки;
- требует статус «Данные уточняются» в каталоге;
- сохраняет `noindex,follow`.

Если какой-либо claim станет `publication_allowed=true`, validator намеренно остановит CI. После этого требуется вручную определить, где и как подтверждённое значение может быть показано.

## Команда

```bash
npm run validate:prostornaya-page-safety
```

Проверка также входит в:

```bash
npm run validate:verification
npm run validate
```
