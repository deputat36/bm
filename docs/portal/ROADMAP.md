# ROADMAP портала novostroyki-borisoglebsk.ru

Дата обновления: 2026-07-09

## Цель дорожной карты

Пошагово превратить старый проект, ориентированный на один объект, в полноценный городской портал по всем новостройкам Борисоглебска.

Главная цель портала:

```text
получать органический трафик и качественные заявки на покупку квартир в разных новостройках Борисоглебска
```

## Этап 1. Управляющий контур

Статус: в работе.

Задачи:

1. Поддерживать `docs/portal/CONTROL_CENTER.md`.
2. Поддерживать `docs/portal/STATUS.md`.
3. Поддерживать `docs/portal/ROADMAP.md`.
4. Поддерживать `docs/portal/DIALOG_PROMPTS.md`.
5. Фиксировать решения из рабочих диалогов.
6. Не допускать расхождения между SEO, юридической безопасностью, архитектурой и лидогенерацией.

Критерий готовности:

```text
все направления работают по единой стратегии и не конфликтуют друг с другом
```

## Этап 2. Аудит текущего проекта

Статус: следующий технический шаг.

Проверить:

1. `index.html`.
2. `sitemap.xml`.
3. `robots.txt`.
4. `CNAME`.
5. `data/site.json`.
6. `data/projects/index.json`.
7. Страницы старого объекта:
   - `/o-zhk/`;
   - `/kvartiry/`;
   - `/planirovki/`;
   - `/ceny/`;
   - `/dokumenty/`;
   - `/galereya/`;
   - `/zastroyschik/`;
   - `/ipoteka/`;
   - `/faq/`;
   - `/contacts/`.

Результат:

```text
понять, какие страницы можно оставить как legacy, какие перенести в каталог, какие закрыть от индексации, а какие переписать
```

## Этап 3. База объектов

Статус: нужно усилить.

Минимальный набор первой очереди:

1. Просторная 4А / ЖК «Теллерманов сад».
2. Аэродромная 18Г.
3. Сенная 76.
4. Проекты ПСК.
5. Проекты BM Group.
6. Остальные новостройки с 2018 года.

Для каждого объекта нужны поля:

```text
адрес
условное название
застройщик
статус
год сдачи / срок сдачи
этажность
количество квартир
площади
материал стен
документы
источники
статус достоверности
можно ли публиковать
```

Результат:

```text
единая база объектов, из которой строятся каталог, карточки, сравнения и SEO-страницы
```

## Этап 5. Главная страница

Статус: не менять без готовой структуры каталога.

Будущая главная должна содержать:

1. Заголовок о новостройках Борисоглебска.
2. Каталог объектов.
3. Фильтр по статусу, адресу, сроку сдачи, типу квартиры.
4. Блок приоритетных объектов.
5. Сравнение новостроек.
6. Ипотечный блок.
7. Справочник покупателя.
8. Форму подбора квартиры.
9. Дисклеймер о независимом каталоге и проверке данных.

## Этап 7. Лидогенерация

Статус: нужно усилить.

Формы:

1. Получить подборку новостроек.
2. Сравнить объекты.
3. Узнать наличие квартир.
4. Рассчитать ипотеку.
5. Получить планировки.
6. Записаться на консультацию.

Минимальные поля:

```text

телефон
интересующий объект

нужна ли ипотека

UTM
согласие на обработку данных
```

## Этап 8. Юридическая безопасность

Статус: обязательный фильтр перед публикацией.

Проверять:

1. Формулировки.
2. Названия застройщиков и ЖК.
3. Упоминания цен и сроков.
4. Обещания брони и наличия квартир.
5. Рекламные посты.
6. SEO-страницы по брендовым запросам.

Запрещённые без проверки формулировки:

```text
официальный сайт
от застройщика
напрямую от застройщика
официальный представитель
цены от застройщика
гарантированная бронь
```

## Этап 9. Публикация

Статус: после готовности каталога и проверки.

Порядок:

1. Подготовить каталог и карточки объектов.
2. Проверить юридические формулировки.
3. Проверить формы заявок.
4. Проверить sitemap и robots.
5. Проверить canonical.
6. Открыть страницы для индексации.
7. Настроить аналитику и цели.
8. Начать продвижение.
