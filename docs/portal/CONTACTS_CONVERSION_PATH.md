# Конверсионный путь страницы контактов

Дата: 2026-07-14

## Проблема

Страница `/contacts/` использовала только подробную форму. До первого разговора посетитель должен был обязательно указать:

```text
имя
телефон
объект
комнатность
```

Дополнительно предлагались бюджет, способ и срок покупки. Для посетителя, который пришёл за общей консультацией, такой первый шаг был длиннее, чем на главной, в каталоге и на страницах объектов.

## Реализованный сценарий

На первом экране добавлена короткая форма:

```text
contacts_quick_selection
```

Обязательные поля:

```text
имя
телефон
интересующий объект
```

Подробная форма сохранена ниже:

```text
contacts_priority_selection
```

Она используется посетителями, готовыми указать комнатность, бюджет, способ и срок покупки.

## Путь пользователя

```text
/contacts/
→ короткая заявка
→ первый разговор
→ уточнение параметров специалистом
```

Альтернативный путь:

```text
/contacts/
→ подробная заявка
→ квалифицированное обращение
```

## Мобильный приоритет

Короткий блок содержит:

```text
id="quick-lead"
data-primary-lead
```

Поэтому мобильная панель действий выбирает именно короткую форму.

## Аналитическая разметка

Шапка:

```text
action=quick_selection
placement=contacts_header
```

Первый экран:

```text
action=quick_selection
placement=contacts_hero
```

Переход к подробной форме:

```text
action=detailed_selection
placement=contacts_hero
```

## Юридические ограничения

- заявка не является бронью;
- заявка не фиксирует цену;
- наличие, стоимость и условия покупки подтверждаются отдельно;
- новые цены, сроки и характеристики объектов не добавлялись;
- страница сохраняет `noindex,follow`;
- портал не позиционируется как официальный сайт застройщика.

## Автоматическая проверка

Файл:

```text
tools/validate-contacts-conversion.mjs
```

Команда:

```bash
npm run validate:contacts-conversion
```

Guard проверяет:

- наличие короткой и подробной формы;
- обязательные поля короткой формы;
- отсутствие комнатности, бюджета, способа, срока и комментария в первом шаге;
- список доступных объектов;
- `inputmode="tel"`;
- порядок форм;
- `data-primary-lead`;
- CTA шапки и первого экрана;
- подключение адаптивных стилей и лид-скриптов.

Проверка включена в общий `npm run validate` и отдельным шагом в Portal guards.

## Ограничение QA

Автоматические проверки не заменяют ручную проверку прокрутки, фокуса, клавиатуры и отображения формы на физических Android и iPhone.
