# STATUS портала novostroyki-borisoglebsk.ru

Дата обновления: 2026-07-13

## Текущий продукт

Портал работает как независимый городской каталог новостроек Борисоглебска.

Приоритетные объекты:

```text
Просторная 4А
Аэродромная 18Г
Сенная 76
```

## Реализовано

- главная и основные разделы переведены на нейтральный портальный стиль;
- для трёх приоритетных объектов работают отдельные формы;
- справочный каталог выводит только подтверждённые карточки;
- заявки сохраняют объект, форму, источник и рекламные метки;
- подключена мобильная панель действий;
- созданы проверки форм, данных, ссылок, текстов, брендинга и миграции;
- страницы `/about/`, `/sources/`, `/legal/`, `/privacy/`, `/personal-data-consent/` и `/advertising/` переведены на новый домен и нейтральную навигацию;
- HTML-карта `/karta-sayta/` перестроена по актуальной структуре без ссылок на legacy-разделы;
- создана рабочая карточка `/developers/bm-group/` с явным статусом повторной проверки;
- старая страница `/zayavka/` больше не содержит самостоятельных форм;
- создан отдельный реестр визуальных материалов Просторной 4А;
- галерея переведена на вывод только подтверждённых и разрешённых к публикации файлов;
- создан построчный реестр проверки характеристик Просторной 4А;
- актуальные рабочие, служебные и доверительные страницы зарегистрированы в `data/pages/index.json`.

## Просторная 4А

В основную карточку перенесены:

```text
характеристики дома
рабочие реквизиты документов
дата последней сверки
безопасные ответы на частые вопросы
основная форма catalog_prostornaya_4a_priority_lead
```

Страница сохраняет `noindex,follow` до повторной проверки первичных источников и формы.

### Построчная проверка данных

Реестр:

```text
data/verification/prostornaya-4a.json
```

В нём отдельно контролируются адрес, застройщик, статус, этажность, квартиры, площади, сроки, декларация, разрешение и квартирография.

Текущий режим:

```text
overall_status=requires_recheck
claims=working_copy
sources=reference_required
is_public_ready=false
```

Для перевода характеристики в `confirmed` требуется проверенный источник с заполненной ссылкой. Пока не подтверждены все `critical` claims, карточка обязана оставаться `noindex,follow`.

Цены, наличие, скидки, бронирование, одобрение ипотеки и окончательный внешний вид явно исключены из документной проверки и требуют отдельной актуализации.

## Застройщики

Создана рабочая карточка:

```text
/developers/bm-group/
```

Она:

```text
закрыта от индексации
не позиционируется как официальный сайт компании
не содержит неподтверждённых контактов или реквизитов
связана с карточкой Просторной 4А
содержит перечень данных для повторной проверки
```

Старый URL `/zastroyschiki/bm-group/` переведён в `transition_page`.

## Визуальные материалы

Реестр:

```text
data/media/prostornaya-4a.json
```

В нём зарегистрировано семь унаследованных архитектурных визуализаций. Для всех установлен статус:

```text
verification_status=requires_check
rights_status=unknown
is_public_ready=false
allowed_usage=[]
```

`assets/js/gallery.js` больше не содержит жёстко заданного списка файлов. Скрипт показывает изображение только при одновременном выполнении условий:

```text
verification_status=confirmed
rights_status=cleared
is_public_ready=true
source_reference заполнен
allowed_usage не пуст
```

До проверки источника и права публикации активные страницы портала не должны подключать галерею.

## Legacy-миграция

Реестр:

```text
data/migration/legacy-routes.json
```

Зарегистрирован 21 маршрут четырёх очередей. Все переходные страницы:

```text
закрыты от индексации
не содержат старого домена и прежнего брендинга
не содержат самостоятельных форм
не используют автоматическое перенаправление
содержат ссылку на актуальный раздел
```

Действия классифицируются как:

```text
redirect
retain_content
retire
```

Полезное содержание перенесено из `/dokumenty/`, `/proektnaya-deklaratsiya-36-001139/` и `/faq/`. Проверка галереи остаётся незавершённой на уровне происхождения и прав.

Четвёртая очередь включает:

```text
/zayavka/ → /contacts/
/zastroyschiki/bm-group/ → /developers/bm-group/
```

```bash

npm run validate:leads
npm run validate:legal
npm run validate:trust
npm run validate:links
npm run validate:legacy
npm run validate:media

```

Дополнительный контроль теперь охватывает:

```text
advertising/index.html
karta-sayta/index.html
developers/bm-group/index.html
zayavka/index.html
zastroyschiki/bm-group/index.html
data/media/**
data/verification/**
```

`validate:verification` проверяет совпадение значений в проектном профиле и реестре, обязательные claims, источники, ссылки и невозможность преждевременно открыть карточку для публикации.

## Главные риски

1. Формы ещё не проверены контролируемой тестовой отправкой.
2. Для Просторной 4А не сохранены точные ссылки на первичные документы.
3. Аэродромная 18Г и Сенная 76 остаются в статусе `requires_check`.
4. Справочный каталог пока не наполнен подтверждёнными объектами.
5. Семь изображений галереи требуют проверки происхождения, актуальности и разрешения на публикацию.
6. Для страниц обработки данных ещё нужно определить фактические реквизиты владельца и провести юридическую проверку.
7. Формат серверных редиректов пока не определён.
8. GitHub-коннектор не показывает результат push workflow для последних коммитов.
9. Общий резервный объект в `assets/js/main.js` всё ещё требует отдельной безопасной замены с сохранением полного файла.

## Следующие действия

1. Сохранить точные ссылки на карточку ЕИСЖС, актуальную декларацию и разрешение Просторной 4А.
2. Сверить critical claims по первичным источникам.
3. Получить подтверждение происхождения и прав на семь визуализаций.
4. Проверить мобильные формы и CTA.
5. Провести контролируемый тест формы `/contacts/`.
6. Собрать источники по другим новостройкам города.
7. Определить платформу размещения и правила редиректов.
8. Заменить резервное значение объекта в `main.js` с ЖК одного проекта на общий подбор портала.
