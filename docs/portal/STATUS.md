# STATUS портала novostroyki-borisoglebsk.ru

Дата обновления: 2026-07-15

## Позиционирование

Портал развивается как независимый городской каталог новостроек Борисоглебска.

Портал не является официальным сайтом застройщика, жилого комплекса или отдела продаж.

Основной принцип:

```text
нет подтверждённого источника → нет публичного факта
```

Корневая страница индексируется. Карточки объектов и внутренние разделы сохраняют `noindex,follow` до прохождения источников, юридической проверки и ворот публикации.


Разрешены пробелы, скобки, дефисы и `+`. Проверка структуры номера не подтверждает существование или принадлежность номера.

## Целостность отправки

Включены:

```text
in-flight lock=true
post-success cooldown=30 секунд
cooldown storage=sessionStorage
cooldown data=form_id + timestamp
honeypot=true
safe storage=true
```

Cooldown создаётся только после подтверждённой внешней отправки. Ошибка, blocked, offline и dry-run не блокируют повторную попытку.

## Privacy-контур

### Delivery

Перед отправкой удаляется device fingerprint `user_agent`.

Контакты клиента не сохраняются в offline fallback. Локально допускается не более пяти обезличенных технических квитанций.

### Dry-run

Dry-run хранит только техническое evidence и булевы признаки заполнения. Полный payload и введённые значения не сохраняются.

### URL и CTA

URL атрибуции очищаются по allowlist из 18 параметров. Произвольные query-поля и fragment не сохраняются.

Телефонные и почтовые CTA передаются в аналитику как технические токены:

```text
phone:primary_sales_phone
email:portal_contact
```

### Сроки хранения

```text
контекст страницы благодарности=24 часа
рекламная атрибуция=30 дней
post-submit cooldown retention=5 минут
```

Повреждённые, будущие и устаревшие записи очищаются.

## Аналитика

Реестр событий: schema version 1.3.

События:

```text
lead_cta_click
lead_form_view
lead_form_start
lead_submit
lead_submit_classified
lead_thankyou_view
lead_postsubmit_action
```

Каноническая заявка:

```text
lead_submit where blocked=false
```

Подтверждённая внешняя отправка:

```text
blocked=false AND offline=false
```

`lead_submit_classified` используется только для разреза `primary/detailed` и не складывается с `lead_submit`.

Прямой GA4-вызов, `dataLayer` и Метрика согласованы по `blocked` и `offline`. Контактные данные и `client_fixation_id` в публичные аналитические события не передаются.

Рабочие GA4 DebugView и Метрика не проверены.

## Dry-run и локальный debug

Обычный dry-run:

```text
?lead_test=dry-run&test_ack=1
```

Локальный analytics debug:

```text
?lead_test=dry-run&analytics_test=debug&test_ack=1
```

Он подавляет внешние аналитические каналы и сохраняет только обезличенные события текущей вкладки.

Тестовый режим страницы благодарности требует свежий совпадающий `NB-TEST` ID. Произвольный `dry_run=1` не может подавить обычную аналитику.
