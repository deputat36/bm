```text
novostroyki-borisoglebsk.ru = городской каталог всех новостроек Борисоглебска
```

Коммерческий приоритет:


Остальные новостройки развиваются как справочный каталог без обязательного создания отдельных лидогенерационных страниц.

## Что уже сделано

1. Главная страница больше не является сайтом одного ЖК или одного застройщика.
2. На главной и в каталоге выделены три приоритетных объекта.
3. Для каждого приоритетного объекта создана отдельная карточка и форма заявки.
4. Общие формы позволяют выбрать объект и фиксируют его системный ID.
5. Поддержана подстановка объекта из параметров `object`, `project` или `complex` в URL.
6. Заявки фиксируют страницу, UTM-метки, источник перехода и ID обращения.
7. `sitemap.xml` переведён на `novostroyki-borisoglebsk.ru`.
8. `robots.txt` указывает на sitemap нового домена.
9. В sitemap временно включена только опубликованная главная.
10. Обновлены основные внутренние разделы:

```text
/developers/
/guides/
/news/
/contacts/
/ipoteka/
```

11. Из этих разделов убраны:

```text
брендинг «Этажи»
логотип BM Group
ссылки на /portal-preview/
canonical старого домена
меню одного ЖК
формы, привязанные только к ЖК «Теллерманов сад»
```

12. Новые внутренние разделы зарегистрированы в `data/pages/index.json` со статусом `ready` и `noindex,follow`.
13. Создан аудит `docs/portal/INTERNAL_SECTIONS_AUDIT_2026-07-12.md`.
14. Создан реестр справочных объектов `data/research/reference-projects.json`.
15. В `/catalog/` добавлен автоматический вывод только подтверждённых справочных карточек.
16. Непроверенные объекты программно исключаются из отображения.
17. Создан валидатор `tools/validate-reference-projects.mjs`.
18. Добавлена команда `npm run validate:reference`.

## Справочный каталог

Справочные карточки выводятся только при выполнении всех условий:

```text
commercial_role=reference_catalog
verification_status=confirmed
is_public_ready=true
есть хотя бы один публичный источник
```

Пока подтверждённых карточек нет, пользователь видит безопасное сообщение о наполнении реестра. Приоритетные формы и общий подбор продолжают работать независимо от справочной части.

## Главные риски

1. В старых страницах могут оставаться canonical и брендинг прежнего проекта.
2. Аэродромная 18Г и Сенная 76 нельзя открывать в индекс до проверки данных.
3. Нельзя публиковать неподтверждённые цены, наличие, сроки и планировки.
4. Реальная доставка новых форм ещё не подтверждена контролируемым тестом.
5. Справочный каталог технически готов, но пока не наполнен подтверждёнными объектами.
6. Нужно проверить работу динамической загрузки каталога на опубликованном домене.
