# STATUS портала novostroyki-borisoglebsk.ru

## Текущий статус

```text
novostroyki-borisoglebsk.ru = городской каталог всех новостроек Борисоглебска
```


Остальные новостройки развиваются как справочный каталог без обязательного создания отдельных лидогенерационных страниц.

## Что уже сделано

1. Главная страница больше не является сайтом одного ЖК или одного застройщика.
2. На главной и в каталоге выделены три приоритетных объекта.
3. Для каждого приоритетного объекта создана отдельная карточка и форма заявки.
4. Общие формы позволяют выбрать объект и фиксируют его системный ID.
5. Поддержана подстановка объекта из параметров `object`, `project` или `complex` в URL.
6. Заявки фиксируют страницу, UTM-метки, источник перехода и ID обращения.
7. `sitemap.xml` переведён на `novostroyki-borisoglebsk.ru`.
8. `robots.txt` указывает на sitemap нового домена.
9. В sitemap временно включена только опубликованная главная.
10. Обновлены основные внутренние разделы `/developers/`, `/guides/`, `/news/`, `/contacts/`, `/ipoteka/`.
11. Из основных разделов убраны старый брендинг, логотип BM Group, ссылки на `/portal-preview/`, canonical старого домена и меню одного ЖК.
12. Новые внутренние разделы зарегистрированы в `data/pages/index.json` со статусом `ready` и `noindex,follow`.
13. Создан реестр справочных объектов `data/research/reference-projects.json`.
14. В `/catalog/` добавлен автоматический вывод только подтверждённых справочных карточек.
15. Непроверенные объекты программно исключаются из отображения.
16. Созданы валидаторы приоритетных объектов, справочного каталога, форм, юридических текстов, ссылок и legacy-маршрутов.
17. GitHub Actions контролирует главную, каталог, внутренние разделы, формы, данные, sitemap, robots и переходные страницы.
18. На мобильных устройствах автоматически добавляется нижняя панель с кнопками «Оставить заявку» и «Позвонить».
19. Выбор справочного объекта переносится в общую форму с ID, названием и адресом объекта.
20. Добавлены события аналитики `reference_catalog_action` и `mobile_lead_bar_click`.
21. Создан машинный реестр переходных URL `data/migration/legacy-routes.json`.
22. Первая очередь старых входных страниц переведена в безопасный переходный режим.
23. Во второй очереди обработаны семь дублирующих страниц Просторной 4А.
24. Для второй очереди создан аудит `docs/portal/LEGACY_OBJECT_PAGES_AUDIT_2026-07-13.md`.
25. Валидатор legacy-страниц запрещает старый домен, прежний брендинг, устаревшие лид-формы и автоматические перенаправления.

## Справочный каталог

Справочные карточки выводятся только при выполнении всех условий:

```text
commercial_role=reference_catalog
verification_status=confirmed
is_public_ready=true
есть хотя бы один публичный источник
```

Пока подтверждённых карточек нет, пользователь видит безопасное сообщение о наполнении реестра. Приоритетные формы и общий подбор продолжают работать независимо от справочной части.

При выборе справочной карточки в заявку добавляются:

```text
reference_object_id
reference_object_name
reference_object_address
```

## Legacy-миграция

Первая очередь:

```text
/portal-preview/ → /
/novostroyki/ → /catalog/
/zhk/ → /catalog/
/zastroyschiki/ → /developers/
/spravochnik/ → /guides/
/novosti/ → /news/
```

Вторая очередь, обработано:

```text
/prostornaya-4a/ → /catalog/prostornaya-4a/
/zhk/tellermanov-sad/ → /catalog/prostornaya-4a/
/o-zhk/ → /catalog/prostornaya-4a/
/kvartiry/ → /catalog/prostornaya-4a/
/planirovki/ → /catalog/prostornaya-4a/
/ceny/ → /catalog/prostornaya-4a/
/dokumenty/ → /catalog/prostornaya-4a/
```

Критические исправления:

- удалён canonical старого домена со страницы `/prostornaya-4a/`;
- удалена дублирующая лид-форма со страницы `/zhk/tellermanov-sad/`;
- все семь страниц получили `noindex,follow` и явную ссылку на новую карточку.

Для переходных страниц действуют правила:

```text
noindex,follow
нет старого домена tellermanovsad.ru
нет старого брендинга Этажи или BM Group
нет устаревших форм
есть явная ссылка на целевой раздел
нет meta refresh или JavaScript redirect
нет URL в sitemap.xml
```

Серверные редиректы ещё не включены. Все маршруты остаются `redirect_ready: false` до проверки целевых страниц и платформы размещения.

## Главные риски

1. В оставшихся legacy-страницах объекта могут быть canonical, устаревшие данные и брендинг прежнего проекта.
2. Аэродромная 18Г и Сенная 76 нельзя открывать в индекс до проверки данных.
3. Нельзя публиковать неподтверждённые цены, наличие, сроки и планировки.
4. Реальная доставка новых форм ещё не подтверждена контролируемым тестом.
5. Справочный каталог технически готов, но пока не наполнен подтверждёнными объектами.
6. Нужно проверить динамическую загрузку каталога и мобильную панель на опубликованном домене.
7. Платформа и формат серверных редиректов пока не зафиксированы.
8. Нужно решить, сохранять ли отдельными страницами галерею, инфраструктуру и ход строительства.
