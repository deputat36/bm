# STATUS портала novostroyki-borisoglebsk.ru

Дата обновления: 2026-07-11

## Текущий статус

Главная страница переведена из формата сайта одного ЖК в формат городского портала-каталога.

Текущая стратегия:

```text
novostroyki-borisoglebsk.ru = городской каталог всех новостроек Борисоглебска
```

Коммерческий приоритет проекта:

```text
сбор заявок по Просторной 4А, Аэродромной 18Г и Сенной 76
```

Остальные новостройки развиваются как справочный каталог без обязательного создания отдельных лидогенерационных страниц.

## Что уже сделано

1. Корневая главная больше не является страницей одного застройщика или одного ЖК.
2. На главной выделены три приоритетных объекта.
3. Для каждого приоритетного объекта есть отдельная карточка и отдельная форма заявки.
4. В формах собираются:
   - имя;
   - телефон;
   - интерес покупателя;
   - бюджет;
   - способ покупки;
   - срок покупки;
   - комментарий.
5. Заявки фиксируют страницу, объект, UTM-метки и источник перехода.
6. Для общей формы добавлено сопоставление названия объекта с его системным ID.
7. Поддержана предварительная подстановка объекта через параметры `object`, `project` или `complex` в URL.
8. Каталог разделён на:
   - три приоритетных объекта;
   - остальные новостройки в справочном формате.
9. Создана рабочая база объектов в `data/projects/`.
10. Созданы документы управления проектом в `docs/portal/`.
11. `sitemap.xml` переведён на домен `novostroyki-borisoglebsk.ru`.
12. `robots.txt` теперь указывает на sitemap нового домена.
13. В sitemap временно оставлена только подтверждённая опубликованная главная страница.

## Остальные новостройки

Другие объекты Борисоглебска добавляются в общий каталог.

Минимальный формат:

```text
адрес
застройщик
статус
год строительства или срок сдачи
основные характеристики
источники
дата проверки
```

Для них не требуется отдельная форма заявки, пока нет подтверждённого спроса или коммерческого приоритета.

## Главные риски

1. Часть старых внутренних страниц относится к прежней архитектуре одного ЖК.
2. Черновые карточки Аэродромной 18Г и Сенной 76 нельзя открывать для индексации до проверки данных.
3. Нельзя публиковать неподтверждённые цены, наличие, сроки, планировки и условия бронирования.
4. Нужно проверить реальную доставку заявок со всех новых форм.
5. Нужно устранить остатки старого брендинга и старых ссылок во внутренних разделах.
6. Нужно проверить статический валидатор после последних изменений.

## Ближайшие автономные действия

1. Провести статическую проверку новых файлов и ссылок.
2. Проверить все формы трёх приоритетных объектов без создания ложных заявок.
3. Подготовить контролируемый тест доставки одной служебной заявки после согласования адресата.
4. Убрать остатки старой навигации и брендинга со внутренних страниц.
5. Начать справочный реестр остальных новостроек Борисоглебска.
6. Подготовить единый шаблон краткой карточки справочного объекта.
7. После проверки решить, когда открывать `/catalog/` для индексации.

## Ограничения

```text
- не публиковать цены без подтверждения;
- не обещать бронь или фиксацию стоимости;
- не использовать формулировки «официальный сайт», «цены от застройщика», «официальный представитель»;
- не открывать в индекс страницы с неподтверждёнными данными;
- не удалять старые страницы без карты редиректов;
- не менять телефон и адресат заявок без отдельного решения.
```
