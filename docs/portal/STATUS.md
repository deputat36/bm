# STATUS портала novostroyki-borisoglebsk.ru

Дата обновления: 2026-07-11

## Текущий статус

Главная страница переведена из формата сайта одного ЖК в формат городского портала-каталога.

Текущая стратегия:

```text
novostroyki-borisoglebsk.ru = городской каталог всех новостроек Борисоглебска
```

Коммерческий приоритет проекта:

```text
сбор заявок по Просторной 4А, Аэродромной 18Г и Сенной 76
```

Остальные новостройки развиваются как справочный каталог без обязательного создания отдельных лидогенерационных страниц.

## Что уже сделано

1. Корневая главная больше не является страницей одного застройщика или одного ЖК.
2. На главной выделены три приоритетных объекта.
3. Для каждого приоритетного объекта есть отдельная карточка и отдельная форма заявки.
4. В формах собираются:
   - имя;
   - телефон;
   - интерес покупателя;
   - бюджет;
   - способ покупки;
   - срок покупки;
   - комментарий.
5. Заявки фиксируют страницу, объект, UTM-метки и источник перехода.
6. Каталог разделён на:
   - три приоритетных объекта;
   - остальные новостройки в справочном формате.
7. Создана рабочая база объектов в `data/projects/`.
8. Созданы документы управления проектом в `docs/portal/`.

## Остальные новостройки

Другие объекты Борисоглебска добавляются в общий каталог.

Минимальный формат:

```text
адрес
застройщик
статус
год строительства или срок сдачи
основные характеристики
источники
дата проверки
```

Для них не требуется отдельная форма заявки, пока нет подтверждённого спроса или коммерческого приоритета.

## Главные риски

1. `sitemap.xml` и `robots.txt` ещё могут содержать старый домен `tellermanovsad.ru`.
2. Часть старых внутренних страниц относится к прежней архитектуре одного ЖК.
3. Черновые карточки Аэродромной 18Г и Сенной 76 нельзя открывать для индексации до проверки данных.
4. Нельзя публиковать неподтверждённые цены, наличие, сроки, планировки и условия бронирования.
5. Нужно проверить реальную доставку заявок со всех новых форм.
6. Нужно устранить остатки старого брендинга и старых ссылок во внутренних разделах.

## Ближайшие автономные действия

1. Создать и поддерживать `LEADS_AND_CRM.md`.
2. Проверить все формы трёх приоритетных объектов.
3. Проверить `spasibo/` и фиксацию объекта после отправки.
4. Подготовить новый sitemap-preview для домена `novostroyki-borisoglebsk.ru`.
5. Проверить и обновить `robots.txt` после готовности sitemap.
6. Убрать остатки старой навигации и брендинга со внутренних страниц.
7. Начать справочный реестр остальных новостроек Борисоглебска.

## Ограничения

```text
- не публиковать цены без подтверждения;
- не обещать бронь или фиксацию стоимости;
- не использовать формулировки «официальный сайт», «цены от застройщика», «официальный представитель»;
- не открывать в индекс страницы с неподтверждёнными данными;
- не удалять старые страницы без карты редиректов;
- не менять телефон и адресат заявок без отдельного решения.
```
