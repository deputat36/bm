# Операционный gate запуска портала

Дата обновления: 2026-07-16

## Назначение

Подготовленные формы и рекламные ссылки не означают, что портал готов принимать реальный трафик. До запуска должно быть определено, кто принимает обращения, когда действует SLA и где фиксируется результат контакта.

Launch-readiness использует отдельное производное ворото:

```text
lead_operations_approval
```

Оно строится из:

```text
data/operations/lead-operations-approval.json
```

## Условия прохождения

Gate становится `passed` только одновременно при выполнении условий:

- в реестре ровно восемь обязательных решений;
- все восемь имеют статус `approved`;
- нет решений `requires_owner_decision`;
- нет решений `rejected`;
- `operational_activation_enabled=true`.

Одного назначения ответственного недостаточно. Необходимо утвердить весь пакет:

1. основной владелец;
2. резервный владелец;
3. рабочий график и часовой пояс;
4. SLA первого действия;
5. правила маршрутизации;
6. политика попыток связи;
7. причины закрытия;
8. единая система учёта.

## Текущее состояние

```text
approved=0
pending=8
activation_enabled=false
status=blocked
```

Это ожидаемое состояние до решений владельца.

## Связь с рекламным запуском

Профиль `campaign_launch` теперь требует:

```text
form_manual_qa
lead_operations_approval
real_lead_delivery
live_analytics_debug
legal_owner_review
campaign_links_prepared
campaign_publication_approval
```

Даже если формы, аналитика и рекламные ссылки готовы, профиль остаётся `BLOCKED`, пока обработка обращений не утверждена.

## Что не меняется

Изменение не:

- назначает конкретного человека;
- включает CRM или Supabase;
- отправляет заявку;
- меняет Web3Forms;
- активирует автоматическую маршрутизацию;
- публикует рекламу;
- сохраняет персональные данные в GitHub.

## Проверка

Команда:

```bash
npm run validate:launch-operations-gate
```

Validator подтверждает:

- наличие нового gate в отчёте;
- включение gate в профиль `campaign_launch`;
- текущие 0 из 8 утверждённых решений;
- выключенную операционную активацию;
- 11 ворот launch-readiness;
- 1 пройденное и 10 заблокированных ворот;
- 0 готовых профилей.

## Изменение статуса в будущем

При утверждении решений нельзя просто поменять итоговый статус отчёта вручную. Сначала обновляется approval-реестр с разрешёнными безопасными значениями и evidence, затем отдельно включается операционная активация. После этого отчёт пересчитывает gate автоматически.
