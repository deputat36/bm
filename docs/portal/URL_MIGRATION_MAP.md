# Карта миграции URL портала novostroyki-borisoglebsk.ru

Дата фиксации: 2026-07-09

Репозиторий: `deputat36/bm`.

## 1. Назначение карты

Этот документ фиксирует, как постепенно перевести проект от старой структуры сайта одного ЖК к структуре городского портала по всем новостройкам Борисоглебска.

Главное правило:

```text
сначала готовим новую структуру в noindex, потом проверяем данные и формы, и только после этого меняем корень сайта, sitemap, robots и canonical
```

## 2. Текущее состояние

Сейчас проект содержит два контура:

1. Старый публичный контур — страницы ЖК «Теллерманов сад» / Просторная 4А.
2. Новый портальный контур — `/catalog/`, `/developers/`, `/guides/`, `/news/` и карточки объектов.

Корневой `index.html` пока относится к старому объектному контуру и не должен заменяться без отдельного подтверждения владельца.

## 4. Основные правила миграции

1. Старые страницы не удалять до финального решения.
2. Новые страницы сначала держать в `noindex,follow`.
3. Не менять рабочий `sitemap.xml` до готовности новой главной.
4. Не менять `robots.txt` до готовности новой структуры.
5. Не менять canonical на старых страницах массово.
6. Не делать редиректы, пока новая структура не проверена.
7. Не открывать в индекс страницы с неподтверждёнными данными.
8. Не публиковать цены, планировки и сроки без источников.

## 8. Когда можно менять sitemap.xml

Рабочий `sitemap.xml` можно заменить только когда:

1. главная портала готова;
2. каталог готов;
3. карточки объектов первой очереди готовы;
4. страницы прошли юридическую проверку;
5. страницы можно открывать для индексации;
6. нет неподтверждённых данных, поданных как факт.

## 9. Когда можно менять robots.txt

`robots.txt` можно менять после подготовки нового sitemap.

Будущий вариант должен указывать:

```text
Sitemap: https://novostroyki-borisoglebsk.ru/sitemap.xml
```

Но до финальной миграции этот файл не трогать без отдельного подтверждения.
