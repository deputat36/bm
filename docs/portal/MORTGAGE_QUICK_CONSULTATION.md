# Короткая ипотечная консультация

Дата: 2026-07-14

## Проблема

Страница `/ipoteka/` использовала только подробную форму. До первого разговора посетитель должен был обязательно указать:

```text
имя
телефон
объект
ипотечную программу или способ покупки
```

Для посетителя, который ещё не знает подходящую программу, обязательный выбор создавал лишний барьер.

## Реализованный сценарий

На первом экране добавлена короткая форма:

```text
portal_mortgage_quick_consultation
```

Обязательные поля:

```text
имя
телефон
интересующий объект
```

Программа, первоначальный взнос, комфортный платёж и срок покупки уточняются в разговоре.

Подробная форма сохранена ниже:

```text
portal_mortgage_consultation
```

В ней по-прежнему обязательно указывается программа или способ покупки.

## Путь пользователя

Короткий сценарий:

```text
объект или раздел ипотеки
→ короткая ипотечная заявка
→ первый разговор
→ уточнение программы и финансовых параметров
```

Подробный сценарий:

```text
раздел ипотеки
→ подробная форма
→ программа + объект + исходные данные
→ подготовленный расчёт
```

## Сохранение объекта

Существующий механизм `assets/js/priority-leads.js` применяется к обеим формам.

При переходе по адресу:

```text
/ipoteka/?object=<residential_complex_id>#quick-lead
```

объект:

- выбирается в короткой форме;
- выбирается в подробной форме;
- записывается в `residential_complex_id`;
- показывается в безопасном уведомлении;
- не подтверждает цену, наличие или применимость программы.

## Мобильный приоритет

Короткий блок содержит:

```text
id="quick-lead"
data-primary-lead
```

Мобильная панель действий выбирает короткую ипотечную форму.

## Аналитическая разметка

Шапка:

```text
action=mortgage_quick_consultation
placement=mortgage_header
```

Короткий CTA первого экрана:

```text
action=mortgage_quick_consultation
placement=mortgage_hero
```

Переход к подробной форме:

```text
action=mortgage_detailed_consultation
placement=mortgage_hero
```

## Юридические ограничения

- предварительная консультация не является одобрением банка;
- не гарантируется ставка или применимость программы;
- заявка не является бронью квартиры;
- заявка не фиксирует стоимость;
- цена и наличие подтверждаются отдельно;
- страница сохраняет `noindex,follow`;
- портал не позиционируется как официальный сайт банка или застройщика.

## Автоматическая проверка

Файл:

```text
tools/validate-mortgage-conversion.mjs
```

Команда:

```bash
npm run validate:mortgage-conversion
```

Guard проверяет:

- наличие короткой и подробной формы;
- обязательные поля первого шага;
- отсутствие программы, комнатности, бюджета, срока и комментария в короткой форме;
- сохранение обязательной программы в подробной форме;
- список объектов;
- `inputmode="tel"`;
- порядок форм;
- `data-primary-lead`;
- CTA шапки и первого экрана;
- уведомление выбранного объекта;
- работу query-параметра и `residential_complex_id`;
- подключение лид-скриптов.

Проверка включена в общий `npm run validate` и отдельным шагом в Portal guards.

## Ограничение QA

Автоматические проверки не подтверждают фактическое банковское решение, реальную доставку заявки и отображение на физических Android и iPhone.
