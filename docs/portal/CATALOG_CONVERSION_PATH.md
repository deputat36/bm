# Конверсионный путь каталога

Дата обновления: 2026-07-14

## Цель

Каталог должен помогать посетителю выбрать направление без обязательного заполнения подробной анкеты до первого разговора.

## Прежний сценарий

До обновления:

- CTA первого экрана вели к подробной форме внизу страницы;
- форма обязательно запрашивала комнатность;
- карточки трёх объектов вели сразу к подробным формам объектных страниц;
- короткие формы, уже созданные на страницах объектов, фактически обходились.

## Новый сценарий

```text
каталог
→ короткая форма на первом экране
→ имя + телефон + объект
→ разговор со специалистом
```

Альтернативный путь:

```text
карточка объекта
→ короткая форма конкретного объекта
→ разговор со специалистом
```

Подробная форма остаётся ниже для посетителя, который готов сразу указать комнатность, бюджет, способ покупки и комментарий.

## Формы

Короткая форма:

```text
catalog_quick_selection
```

Поля:

```text
name
phone
residential_complex
```

Подробная форма:

```text
catalog_priority_selection
```

Она сохраняет расширенные параметры подбора.

Обе формы используют:

```text
lead_type=portal_selection
project_id=newbuilds-borisoglebsk
```

Выбранный объект синхронизируется с `residential_complex_id` через существующий разрешённый реестр.

## Карточки объектов

CTA карточек ведут на:

```text
/catalog/prostornaya-4a/#quick-lead
/catalog/aerodromnaya-18g/#quick-lead
/catalog/sennaya-76/#quick-lead
```

Каждый CTA содержит:

```text
data-track-action=object_quick_consultation
data-track-placement=catalog_card
data-track-object=<ID объекта>
```

## Мобильная версия

Короткая форма отмечена:

```text
data-primary-lead
```

Поэтому мобильная панель ведёт к первому короткому сценарию, а не к подробной форме внизу.

Телефон использует:

```text
inputmode=tel
autocomplete=tel
```

## Аналитика

Обе формы входят в существующую воронку:

```text
lead_cta_click
lead_form_view
lead_form_start
lead_submit
lead_thankyou_view
```

Сравнение выполняется по `form_id`:

```text
catalog_quick_selection
catalog_priority_selection
```

## Юридическая безопасность

- короткая форма не обещает наличие квартиры;
- заявка не является бронью и не фиксирует цену;
- сведения сообщаются после проверки на дату обращения;
- каталог остаётся `noindex,follow` до финальной проверки;
- новые цены, сроки и характеристики не добавлялись.

## Автоматическая проверка

Команда:

```bash
npm run validate
```

запускает `tools/validate-catalog-conversion.mjs`.

Guard проверяет:

- наличие короткой и подробной форм;
- порядок форм;
- три обязательных поля короткой формы;
- отсутствие бюджета, способа покупки, срока, комментария и комнатности в короткой форме;
- полный список объектов;
- мобильный приоритет;
- CTA первого экрана;
- переход карточек к коротким формам объектов;
- разметку аналитики;
- сохранение подробного сценария.
