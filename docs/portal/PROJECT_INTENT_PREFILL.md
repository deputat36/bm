# Предзаполнение намерения в объектных формах

Дата обновления: 2026-07-16

## Цель

Помочь посетителю перейти к конкретному вопросу без создания дополнительных форм и без передачи значений через URL.

## Маршруты

| Объект | CTA | Предзаполняемое значение | Placement |
|---|---|---|---|
| Просторная 4А | Проверить документы | Документы по объекту | `project_documents_check` |
| Аэродромная 18Г | Проверить схему покупки | Схема покупки и договор | `project_purchase_scheme` |
| Сенная 76 | Проверить конкретную квартиру | Проверка конкретной квартиры | `project_apartment_check` |

## Поведение

После клика:

1. Находится существующий блок `#quick-lead`.
2. Находится primary-форма и `select[name=interest]`.
3. Выбирается только существующий option с точным совпадением.
4. Страница прокручивается к форме.
5. Фокус переводится на select.

Пользователь может изменить выбранный вариант до отправки.

## Аналитика

Каждый CTA использует действующий `lead_cta_click`:

```text
action=intent_prefill
placement=<уникальное размещение>
object_id=<объект>
```

Предзаполненное значение попадёт в заявку только как обычное поле `interest` после отправки формы.

## Privacy

Runtime не использует:

- query-параметры;
- localStorage или sessionStorage;
- cookie;
- network-запросы;
- скрытые поля;
- персональные данные.

Синтетический `change` не отправляется. `lead_form_start` фиксируется после фокуса, вызванного осознанным кликом пользователя.

## Ограничения

- число форм остаётся 14;
- на каждой объектной странице остаётся две формы;
- CTA не является заявкой и не фиксирует цену или квартиру;
- проверка конкретной квартиры не означает подтверждение её наличия или права продавца;
- проверка схемы покупки не является юридическим заключением.

## Guard

Файлы:

```text
tools/validate-project-intent-prefill.mjs
.github/workflows/project-intent-prefill.yml
```

Проверяются:

- три точных CTA и placement;
- наличие option в целевых формах;
- два исходных form_id на каждой странице;
- отсутствие новых форм;
- порядок загрузки runtime;
- отсутствие query, storage, fetch и `innerHTML`;
- сохранение `noindex,follow`.
