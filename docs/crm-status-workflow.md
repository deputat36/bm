# CRM-воронка заявок портала

Документ описывает статусы обработки заявок портала «Новостройки Борисоглебска».

## 1. Цель

Статусы нужны, чтобы менеджер видел не просто новую заявку, а понятный путь обработки:

```text
новая заявка → первый контакт → квалификация → следующий шаг → сделка или потеря
```

## 2. Основные статусы

### new

Заявка только создана с сайта.

Действие менеджера:

- открыть заявку;
- проверить телефон;
- посмотреть тип заявки;
- посмотреть квалификацию `hot`, `warm` или `cold`;
- сделать первый контакт.

### in_work

Менеджер взял заявку в работу.

Действие менеджера:

- позвонить;
- написать сообщение, если звонок не прошёл;
- зафиксировать результат.

### first_contact_done

Первый контакт состоялся.

Действие менеджера:

- уточнить интерес;
- уточнить бюджет;
- уточнить способ покупки;
- определить следующий шаг.

### no_answer

Клиент не ответил.

Действие менеджера:

- зафиксировать попытку;
- поставить следующую дату контакта;
- сделать повторное касание.

### qualified

Покупатель квалифицирован.

Действие менеджера:

- определить подходящие варианты;
- выбрать сценарий: ипотека, подбор, ожидание, встреча.

### mortgage_check

Нужно проверить ипотечный сценарий.

Действие менеджера:

- уточнить первоначальный взнос;
- уточнить доход;
- уточнить семейную ипотеку, маткапитал или сертификаты;
- передать ипотечному специалисту, если нужно.

### selection

Нужен подбор квартиры или объекта.

Действие менеджера:

- подобрать варианты;
- отправить подборку;
- согласовать следующий контакт.

### waiting_prices

Покупатель ждёт подтверждённые цены, планировки или старт продаж.

Действие менеджера:

- добавить в список ожидания;
- указать, что именно сообщить;
- не обещать цену до подтверждения.

### meeting_planned

Назначена встреча или консультация.

Действие менеджера:

- указать дату и время;
- подготовить материалы;
- напомнить клиенту перед встречей.

### deal_started

Началась сделочная работа.

Действие менеджера:

- перевести работу в сделочный контур;
- зафиксировать ответственных;
- связать заявку с объектом и клиентом.

### lost

Заявка потеряна.

Действие менеджера:

- указать причину потери;
- не удалять историю;
- оставить возможность повторного касания.

### duplicate

Повторная заявка от того же клиента.

Действие менеджера:

- связать с основной карточкой;
- не создавать конфликт по ответственному;
- сохранить новый источник обращения.

### spam

Заявка признана технической или мусорной.

Действие менеджера:

- не обрабатывать как клиента;
- сохранить технические признаки;
- при повторении передать администратору.

## 3. Рекомендуемые переходы

```text
new → in_work
in_work → first_contact_done
in_work → no_answer
first_contact_done → qualified
qualified → mortgage_check
qualified → selection
qualified → waiting_prices
qualified → meeting_planned
meeting_planned → deal_started
new → duplicate
new → spam
любая рабочая стадия → lost
```

## 4. Что обязательно фиксировать

После каждого контакта фиксировать:

- дату и время действия;
- кто обработал заявку;
- результат контакта;
- следующий шаг;
- дату следующего контакта;
- интересующий объект;
- бюджет;
- способ покупки;
- комментарий менеджера.

## 5. Причины потери

Рекомендуемые причины:

- не дозвонились после нескольких попыток;
- передумал покупать;
- не подходит бюджет;
- не подходит срок сдачи;
- выбрал другой объект;
- купил через другой канал;
- нет подходящей программы покупки;
- некорректный номер;
- дубль;
- техническая заявка.

## 6. SLA обработки

### hot

Первый контакт — как можно быстрее в рабочее время.

### warm

Первый контакт — в течение рабочего дня.

### cold

Первое касание — без потери заявки, затем прогрев.

## 7. Безопасные формулировки

Использовать:

- «обращение зафиксировано»;
- «интерес зафиксирован»;
- «добавили в список ожидания»;
- «сообщим после появления подтверждённых данных»;
- «условия нужно уточнить перед решением».

Не использовать:

- формулировки, которые обещают цену;
- формулировки, которые обещают наличие квартиры;
- формулировки, которые создают впечатление официального подтверждения от застройщика.
