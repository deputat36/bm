# Проверка типов заявок в валидаторе

В `tools/validate-static-site.mjs` добавлена проверка типов заявок.

## 1. Что проверяется

Валидатор смотрит формы с атрибутом:

```text
data-lead-form
```

И проверяет значение:

```text
data-lead-type
```

## 2. Допустимые типы

Поддерживаются:

```text
complex_interest
mortgage
apartment_selection
consultation
callback
waitlist
portal_selection
project_consultation
general
```

Если форма в новом черновом разделе использует другой тип, это ошибка.

## 3. Согласованность с SQL

Валидатор проверяет, что все допустимые типы есть в файле:

```text
docs/supabase-leads.sql
```

Если тип отсутствует в SQL-схеме, это ошибка.

Причина: база может отклонить заявку по CHECK-ограничению.

## 4. Согласованность с серверной функцией

Валидатор также проверяет файл:

```text
supabase/functions/newbuild-lead/index.ts
```

Если там отсутствует один из допустимых типов, валидатор выдаёт предупреждение.

Это предупреждение нужно исправить до подключения активной серверной отправки заявок.

## 5. Почему это важно

Фронтенд уже использует портальные типы:

```text
portal_selection
project_consultation
```

Если серверная функция не принимает эти значения, заявка может не пройти валидацию.

## 6. Как проверить

Из корня проекта выполнить:

```bash
npm run validate
```

В выводе появится строка:

```text
Checked lead types: ...
```

Если серверная функция ещё не синхронизирована, будет предупреждение с недостающими типами.
