# Чек-лист проверки микроразметки

Документ нужен перед тем, как открывать страницы портала для индексации и добавлять их в `sitemap.xml`.

## 1. Что проверять

Проверять нужно не сам факт наличия JSON-LD, а соответствие микроразметки видимому содержанию страницы.

Главные правила:

- данные в JSON-LD должны совпадать с текстом на странице;
- нельзя добавлять цены, если они не подтверждены;
- нельзя использовать `Offer`, если нет юридически безопасного основания;
- нельзя создавать впечатление официального сайта застройщика;
- лучше меньше микроразметки, но точнее.

## 2. Страница ЖК

Для страницы:

```text
/zhk/tellermanov-sad/
```

Ожидаемая микроразметка:

```text
BreadcrumbList
Residence
```

Проверить:

- хлебные крошки ведут на реальные страницы;
- последняя часть хлебных крошек совпадает с названием страницы;
- `Residence.name` соответствует видимому названию ЖК;
- адрес в `Residence.address` соответствует видимому адресу;
- этажность соответствует видимому тексту;
- нет `Offer`, цены или обещания наличия квартир.

## 3. Главная портала

Для будущей корневой главной после переноса `/portal-preview/` ожидается:

```text
WebSite
```

Проверить:

- `WebSite.name` — «Новостройки Борисоглебска»;
- `WebSite.description` описывает портал, а не официальный сайт ЖК;
- URL ведёт на корневую страницу портала;
- нет разметки застройщика как владельца сайта;
- нет `Offer` и неподтвержденных коммерческих условий.

## 4. Каталог новостроек

Для будущих страниц:

```text
/novostroyki/
/zhk/
```

Планируемая микроразметка:

```text
BreadcrumbList
ItemList
```

Проверить:

- каждый объект из `ItemList` реально есть на странице;
- название объекта совпадает с карточкой;
- ссылка ведёт на карточку объекта;
- статус данных не противоречит видимому тексту;
- не указаны неподтвержденные цены.

## 5. Застройщик

Для будущей страницы:

```text
/zastroyschiki/bm-group/
```

Планируемая микроразметка:

```text
BreadcrumbList
Organization
```

Проверить:

- юридическое лицо совпадает с видимым текстом;
- не создаётся впечатление, что портал является официальным сайтом застройщика;
- связи с объектами указаны только для проверенных объектов;
- контакты добавляются только при наличии безопасного основания.

## 6. FAQ

Для будущих FAQ-страниц возможна микроразметка:

```text
FAQPage
```

Проверить:

- каждый вопрос и ответ виден на странице;
- ответы не обещают цену, бронь или одобрение ипотеки;
- формулировки юридически безопасны;
- в ответах нет неподтвержденных данных.

## 7. Проверка в браузере

Открыть страницу и выполнить в консоли:

```js
Array.from(document.querySelectorAll('script[type="application/ld+json"]')).map((item) => JSON.parse(item.textContent));
```

Проверить, что массив содержит ожидаемые типы.

## 8. Внешняя проверка

После публикации проверить страницу через валидаторы:

- Rich Results Test;
- Schema Markup Validator;
- Яндекс.Вебмастер после индексации.

## 9. Когда не подключать микроразметку

Не подключать микроразметку, если:

- страница ещё черновая;
- данные не сверены;
- нет понимания canonical;
- контент отличается от data-атрибутов;
- на странице используются временные тексты;
- есть риск юридически неверного позиционирования.
