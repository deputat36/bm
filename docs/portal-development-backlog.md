# Backlog разработки портала «Новостройки Борисоглебска»

Дата: 05.07.2026  
Проект: `novostroyki-borisoglebsk.ru`  
Назначение: превратить SEO-стратегию и карту URL в очередность конкретных задач для разработки.

## 1. Подход к разработке

Разработку нужно вести небольшими безопасными шагами.

Основная логика:

1. Сначала подготовить архитектуру и базовые шаблоны.
2. Затем создать городские хабы.
3. Потом перенести ЖК «Теллерманов сад» в структуру портала.
4. После этого расширять семантику коммерческими и информационными страницами.
5. На каждом этапе проверять индексацию, canonical, sitemap, формы и юридические формулировки.

## 2. Этап 0 — подготовка

### Задача 0.1. Зафиксировать доменную стратегию

Решить, какой домен будет главным:

- `novostroyki-borisoglebsk.ru` — городской портал;
- `tellermanovsad.ru` — отдельный лендинг ЖК или редирект на раздел портала.

Рекомендуемый вариант:

- портал развивать как главный домен;
- `tellermanovsad.ru` временно оставить, пока портал не готов;
- после запуска решить: оставить лендинг или перевести его 301-редиректом на `/zhk/tellermanov-sad/`.

Критерии готовности:

- принято решение по canonical;
- понятно, какие страницы будут индексироваться на каждом домене;
- нет конкуренции одинаковых текстов между доменами.

### Задача 0.2. Проверить текущие публичные страницы

Нужно составить фактический список текущих HTML-страниц и разделов.

Проверить:

- главную;
- разделы ЖК;
- новости;
- служебные страницы;
- sitemap;
- robots;
- формы заявок;
- UTM и параметры риэлтора.

Критерии готовности:

- есть список текущих URL;
- каждому URL назначено действие: оставить, перенести, редирект, закрыть, объединить;
- нет страниц с непонятным назначением.

Критерии готовности:

- данные не зашиты хаотично в каждую страницу;
- у ЖК есть единый slug;
- на всех страницах можно выводить дату обновления;
- структура пригодна для второго ЖК в будущем.

### Задача 1.2. Создать новый header портала

Главное меню:

- Новостройки;
- Квартиры;
- Планировки;
- Цены;
- Ипотека;
- Застройщики;
- Районы;
- Блог;
- Контакты.

Кнопка:

- Подобрать квартиру.

Критерии готовности:

- меню не выглядит как сайт одного застройщика;
- нет визуального ощущения «официального сайта ЖК»;
- на мобильном меню не ломается;
- все основные хабы доступны в один клик.

### Задача 1.3. Создать новый footer портала

Footer должен содержать:

- название портала;
- краткое описание;
- меню портала;
- популярные разделы;
- контакты;
- политику конфиденциальности;
- юридический дисклеймер.

Дисклеймер:

> Портал является информационным городским каталогом новостроек Борисоглебска и не является официальным сайтом застройщика, если это прямо не указано на странице конкретного объекта. Информация обновляется по мере появления подтвержденных данных.

Критерии готовности:

- пользователь понимает, что это городской портал;
- юридическая позиция аккуратная;
- в footer есть ссылки на все SEO-хабы.

## 5. Этап 3 — техническое SEO

### Задача 3.1. Обновить sitemap

Сделать sitemap под `novostroyki-borisoglebsk.ru`.

Критерии готовности:

- sitemap содержит только индексируемые страницы;
- URL соответствуют новой структуре;
- lastmod обновлен;
- нет старых URL `tellermanovsad.ru`, если работаем под новым доменом.

### Задача 3.2. Обновить robots.txt

Нужно заменить `Sitemap: https://tellermanovsad.ru/sitemap.xml` на новый домен после запуска портала.

Критерии готовности:

- служебные разделы закрыты;
- коммерческие хабы не закрыты;
- sitemap указывает на новый домен.

### Задача 3.3. Добавить canonical

Критерии готовности:

- на каждой странице canonical указывает на чистый основной URL;
- нет canonical на старый домен, если страница уже часть портала;
- страницы с параметрами UTM не создают дублей.

### Задача 3.4. Добавить микроразметку

Минимум:

- `Organization`;
- `BreadcrumbList`;
- `FAQPage`;
- `Residence` или `ApartmentComplex` для ЖК;
- `Article` для статей.

Критерии готовности:

- разметка валидна;
- не указаны неподтвержденные цены как `Offer`;
- FAQ соответствует видимому тексту на странице.

## 9. Риски

### Риск 1. Дубли между доменами

Если одинаковые тексты будут на `tellermanovsad.ru` и `novostroyki-borisoglebsk.ru`, страницы начнут конкурировать.

Решение:

- выбрать основной домен;
- использовать canonical или 301;
- уникализировать тексты при временной работе двух доменов.

### Риск 2. Пустые SEO-страницы

Страницы по типам квартир, бюджету и районам могут быть пустыми, если данных мало.

Решение:

- запускать только наполненные страницы;
- слабые страницы держать в черновиках;
- не добавлять их в sitemap до готовности.

### Риск 3. Юридическое позиционирование

Портал нельзя представить как официальный сайт застройщика, если это не так.

Решение:

- аккуратный дисклеймер;
- ссылки на источники;
- формулировки «информационный городской каталог»;
- не использовать запрещенные формулировки из договоров и рекламных ограничений.

### Риск 4. Потеря текущего SEO по ЖК

При резком переносе можно потерять уже занятые позиции.

Решение:

- миграция через 301;
- сохранение структуры смыслов;
- обновление sitemap;
- контроль индексации;
- постепенное усиление страниц ЖК внутри портала.
