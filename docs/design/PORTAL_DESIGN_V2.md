# Новостройки Борисоглебска — Design System v2

## Figma

Файл проекта: https://www.figma.com/design/rhFYa5gPDhF009hZsfEGSX

Название концепции: **«Городской навигатор»**.

Figma Starter временно блокирует MCP-чтения и записи по лимиту. До восстановления доступа код и `data/design/portal-v2.tokens.json` являются источником правды. После открытия лимита переменные, компоненты и экраны должны быть перенесены в Figma без изменения значений.

### Подготовленный Foundations handoff

Для автоматического создания нативных Figma Variables, Text Styles и Effect Styles подготовлены:

- `assets/css/portal-v2.tokens.css` — WEB Code Syntax и CSS-контракт;
- `tools/figma/generate-portal-v2-foundations.mjs` — генератор идемпотентного `use_figma`-кода;
- `tools/validate-figma-foundations-handoff.mjs` — проверка aliases, scopes, CSS и Plugin API-кода;
- `docs/design/FIGMA_FOUNDATIONS_HANDOFF.md` — инструкция запуска и критерии проверки;
- `.github/workflows/figma-foundations-handoff.yml` — отдельный CI-guard.

Semantic colors имеют явную связь с primitive colors через `semantic_aliases`. Это исключает raw HEX в semantic collection и позволяет менять тему из одного primitive-слоя.

## Живая витрина

В репозитории есть закрытая браузерная страница:

`/design-system/`

Она использует production CSS и показывает:

- палитру и типографическую шкалу;
- основные и вторичные кнопки;
- бейджи и статусы;
- карточки ЖК и карточки фактов;
- демонстрационную форму без отправки;
- подтверждённый и ожидающий проверки статусы;
- FAQ;
- мобильную нижнюю панель действий.

Страница имеет `noindex,nofollow`, не включается в sitemap, не содержит `<form>`, `data-lead-form`, JavaScript отправки или кнопок `type="submit"`. Она нужна для визуального QA, передачи дизайна и будущего импорта в Figma.

Связанные файлы:

- `design-system/index.html`;
- `assets/css/design-system-preview.css`;
- `tools/validate-design-system-preview.mjs`;
- `.github/workflows/design-system-preview.yml`.

## Позиционирование

Портал выглядит как независимый городской сервис выбора и проверки новостроек, а не как официальный сайт одного застройщика или рекламная посадочная страница.

Основные ощущения:

- спокойная экспертность;
- понятный выбор следующего шага;
- визуальное разделение подтверждённых данных и сведений, которые требуют проверки;
- крупные формы и кнопки без агрессивного давления;
- удобное чтение на телефоне.

## Визуальный язык

### Цвет

- тёмно-синий — доверие, навигация, основные заголовки и тёмные секции;
- коралловый — главный CTA и активное действие;
- янтарный — внимание, фокус и ожидающие проверки статусы;
- зелёный — подтверждённые сведения и безопасные статусы;
- холодные светло-серые поверхности — каталог и информационные разделы.

Точные значения хранятся в `data/design/portal-v2.tokens.json`.

### Типографика

Используется системный стек без внешней загрузки шрифтов:

`system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", Arial, sans-serif`

Причины:

- нет блокировки отображения текста;
- высокая скорость загрузки;
- корректная кириллица на Windows, Android, macOS и iOS;
- отсутствие внешней зависимости и проблем с лицензией.

Заголовки используют вес 900. Основной текст — 400–500. Служебные бейджи — 800–900.

## Компоненты v1

1. Brand — самостоятельный знак портала и текстовое название.
2. Top navigation — светлая липкая панель с нейтральными ссылками и одним CTA.
3. Button primary — коралловый, для консультации и начала подбора.
4. Button secondary — белый или тёмно-синий, для просмотра и сравнения.
5. Eyebrow badge — контекст раздела или статус.
6. Project card — карточка объекта, преимущества и безопасные CTA.
7. Fact card — крупный факт с пояснением и источником.
8. Lead form card — белая плавающая карточка с цветовой полосой доверия.
9. Input / Select / Textarea — высота не менее 50 px и заметный focus ring.
10. Verification status — зелёный подтверждённый и янтарный ожидающий блок.
11. FAQ accordion — крупная кликабельная строка и явное состояние раскрытия.
12. Mobile action bar — звонок и заявка в нижней зоне большого пальца.

## Адаптивность

- desktop container: 1200 px;
- tablet breakpoint: 980 px;
- mobile breakpoint: 640 px;
- минимальная высота интерактивного элемента: 48 px;
- на мобильном основные CTA занимают всю ширину;
- нижняя панель сохраняет звонок и заявку;
- сетки 3–4 колонки переходят в 2, затем в 1 колонку.

## Что нельзя менять при визуальных обновлениях

- `data-form-id`, `data-lead-type`, `data-complex-id` и другие идентификаторы заявок;
- порядок и смысл полей согласия;
- юридические предупреждения;
- `noindex,follow` на непроверенных страницах;
- атрибуцию источников;
- tracking attributes;
- существующие URL и якоря.

## Реализация

Основной слой: `assets/css/home-polish.css`.

Специализированные слои:

- `assets/css/portal-v2.tokens.css` — машиночитаемый контракт кода и Figma;
- `assets/css/leadgen.css` — формы и лидогенерация;
- `assets/css/project-conversion.css` — карточки ЖК, статусы и FAQ;
- `assets/css/design-system-preview.css` — только закрытая живая витрина.

Базовый `assets/css/styles.css` оставлен как совместимый структурный слой. Это уменьшает риск регрессий и позволяет поэтапно удалить устаревшие правила после визуальной проверки всех страниц.
