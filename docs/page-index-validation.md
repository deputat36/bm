# Проверка индекса страниц

Дата фиксации: 2026-07-07

## 1. Назначение

Файл `data/pages/index.json` нужен как рабочий реестр страниц портала. В дальнейшем он может использоваться для подготовки sitemap, проверки canonical, контроля индексации и понимания статуса каждой страницы.

## 2. Что теперь проверяет валидатор

Команда:

```text
npm run validate
```

проверяет не только HTML, формы и JSON-данные проектов, но и `data/pages/index.json`.

Для каждой записи проверяется:

- URL начинается с `/`;
- URL не дублируется;
- указано название страницы `title`;
- указан тип страницы `page_type`;
- статус входит в список `draft`, `ready`, `published`, `archived`;
- для объектных страниц указан `project_id`;
- HTML-файл по URL существует;
- если в индексе указано `robots = noindex,follow`, в HTML есть такой же meta robots;
- страница со статусом `published` не может иметь `robots = noindex,follow`.

## 3. Почему это важно

Без этой проверки можно случайно:

- добавить страницу в реестр, но не создать HTML-файл;
- забыть `noindex` на черновой странице;
- пометить страницу как опубликованную, но оставить закрытой от индексации;
- потерять связь объектной страницы с ЖК;
- подготовить неправильный sitemap после миграции.

## 4. Текущий принцип

Все новые разделы портала пока должны оставаться в статусе:

```text
status = draft
robots = noindex,follow
```

Перевод в `published` допустим только после ручной проверки страницы, данных, форм, юридических формулировок и стратегии миграции.
