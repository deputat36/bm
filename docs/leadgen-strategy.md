# Лидогенерация портала «Новостройки Борисоглебска»

Документ описывает лидогенерационный контур портала: формы, квалификацию покупателя, UTM-метки, передачу заявок, сценарии обработки менеджером и юридически безопасную фиксацию обращения.

## 1. Цель лидогенерации

Главная цель — получать не просто телефоны, а структурированные обращения покупателей новостроек Борисоглебска:

- покупатель конкретного ЖК;
- ипотечный покупатель;
- покупатель, которому нужен подбор;
- человек с вопросом;
- человек, который просит перезвонить;
- человек из списка ожидания.

Каждая заявка должна отвечать на вопросы:

1. Откуда пришёл человек?
2. Какая форма сработала?
3. Какой ЖК или тип квартиры интересует?
4. Есть ли бюджет, ипотека, срок покупки?
5. Насколько срочно менеджеру нужно обрабатывать заявку?
6. На каком основании фиксируется обращение клиента?

## 2. Реализованные формы

Страница: `/zayavka/`.

### 2.1. Заявка на квартиру в ЖК

Тип заявки: `complex_interest`.

Задача формы — собрать тёплый спрос по конкретному жилому комплексу.

Поля:

- имя;
- телефон;
- ЖК;
- комнатность;
- бюджет;
- способ покупки;
- комментарий.

Когда использовать:

- рекламные посты по ЖК;
- кнопки «получить цены»;
- карточки квартир;
- страницы планировок;
- посадочные страницы по конкретному ЖК.

### 2.2. Ипотечная консультация

Тип заявки: `mortgage`.

Задача формы — собрать обращение, где покупателю нужно посчитать платёж, взнос или программу.

Поля:

- имя;
- телефон;
- ипотечная программа;
- первоначальный взнос;
- комфортный платёж;
- подтверждение дохода;
- комментарий.

Когда использовать:

- страница ипотеки;
- рекламные объявления по семейной ипотеке;
- посты про материнский капитал;
- сценарии «проверьте, проходите ли вы по программе».

### 2.3. Подбор квартиры

Тип заявки: `apartment_selection`.

Задача формы — получить параметры покупателя, который ещё выбирает между вариантами.

Поля:

- имя;
- телефон;
- комнатность;
- бюджет;
- срок покупки;
- есть ли своя недвижимость на продажу;
- пожелания.

Когда использовать:

- общие страницы портала;
- статьи «как выбрать квартиру»;
- страницы с подборками;
- рекламные кампании без привязки к одному ЖК.

### 2.4. Консультация специалиста

Тип заявки: `consultation`.

Задача формы — собрать вопрос и подготовить менеджера к разговору.

Поля:

- имя;
- телефон;
- тема вопроса;
- удобное время связи;
- вопрос.

Когда использовать:

- FAQ;
- документы;
- статьи про безопасность сделки;
- страницы застройщика и проектной декларации.

### 2.5. Заказать звонок

Тип заявки: `callback`.

Задача формы — максимально быстрый вход для человека, который не хочет заполнять длинную анкету.

Поля:

- имя;
- телефон;
- удобное время звонка;
- тема;
- комментарий.

Когда использовать:

- плавающие кнопки;
- мобильная версия;
- баннеры;
- быстрые CTA-блоки.

### 2.6. Список ожидания

Тип заявки: `waitlist`.

Задача формы — собрать базу покупателей, которым нужно сообщить о старте продаж, ценах, планировках или появлении квартиры.

Поля:

- имя;
- телефон;
- что сообщить;
- интересующая квартира;
- бюджет;
- канал связи;
- отдельная отметка на информационные сообщения;
- комментарий.

Когда использовать:

- до старта продаж;
- когда цена ещё не подтверждена;
- когда есть дефицит конкретных планировок;
- для прогрева аудитории.

## 3. Квалификация покупателя

Фронтенд автоматически добавляет объект `qualification`.

Статусы:

- `hot` — срочно обработать;
- `warm` — обработать в рабочий день;
- `cold` — добавить в прогрев.

Факторы, которые повышают балл:

- указан корректный телефон;
- понятен интерес по квартире или услуге;
- есть бюджет или финансовый ориентир;
- указан способ покупки;
- указан короткий срок покупки;
- заказан звонок;
- нужна ипотечная квалификация;
- есть встречная продажа.

Рекомендуемая реакция менеджера:

- `hot`: звонок в течение 5–15 минут в рабочее время;
- `warm`: звонок в течение рабочего дня;
- `cold`: сообщение/звонок, добавление в прогрев, повторное касание после появления новостей.

- `utm_term`;

Также сохраняется:

Пример ссылки для рекламы:

```text
https://tellermanovsad.ru/zayavka/?utm_source=vk&utm_medium=post&utm_campaign=tellermanov_start&utm_content=price_waitlist&realtor=kovtun&realtor_id=1
```

```text
https://tellermanovsad.ru/zayavka/#ipoteka?utm_source=vk&utm_medium=post&utm_campaign=family_mortgage
```

Важно: якорь `#ipoteka` лучше ставить после query string:

```text
https://tellermanovsad.ru/zayavka/?utm_source=vk&utm_medium=post&utm_campaign=family_mortgage#ipoteka
```

## 5. Передача заявок

Текущая реализация:

- email-копия через Web3Forms;
- подготовлена возможность подключения `LEAD_ENDPOINT` для CRM, Supabase Edge Function или другого обработчика.

Файл настройки: `assets/js/main.js`.

```js
const SITE_CONFIG = {
  WEB3FORMS_ACCESS_KEY: "...",
  LEAD_ENDPOINT: "",
  SEND_EMAIL_COPY: true
};
```

Правильная схема после подключения Supabase/CRM:

1. Форма отправляет JSON в `LEAD_ENDPOINT`.
2. Endpoint валидирует данные.
3. Endpoint записывает заявку в Supabase/CRM.
4. Endpoint отправляет уведомление менеджеру: email, Telegram, CRM-задача.
5. Web3Forms остаётся как резервная email-копия или отключается.

## 7. Статусы обработки заявки

Рекомендуемые CRM-статусы:

1. `new` — новая заявка;
2. `in_work` — менеджер взял в работу;
3. `first_contact_done` — первый контакт состоялся;
4. `no_answer` — не дозвонились;
5. `qualified` — покупатель квалифицирован;
6. `mortgage_check` — передан на ипотечную консультацию;
7. `selection` — идёт подбор квартиры;
8. `waiting_prices` — ждёт цены / старт продаж;
9. `meeting_planned` — назначена встреча;
10. `booking_interest` — есть интерес к бронированию;
11. `deal_started` — началась сделка;
12. `lost` — отказ / неактуально;
13. `duplicate` — дубль;
14. `spam` — спам.

## 8. Сценарии обработки менеджером

### 8.1. Заявка по ЖК

Цель звонка — подтвердить интерес, бюджет, способ покупки и следующий шаг.

Скрипт:

```text
Здравствуйте, [Имя]. Вы оставляли заявку по новостройке в Борисоглебске, ЖК «Теллерманов сад». Подскажите, вам удобнее сейчас пару минут обсудить, какая квартира вам нужна?
```

Ключевые вопросы:

1. Какая комнатность интересует?
2. Для себя, детей, родителей или инвестиции?
3. Какой бюджет рассматриваете?
4. Покупка наличными или с ипотекой?
5. Есть ли своя недвижимость на продажу?
6. Когда планируете принимать решение?
7. Что важно: этаж, площадь, вид, сторона, цена, платёж?

Финал:

```text
Я зафиксировал ваш интерес. Как только появятся подтверждённые цены и условия, я сообщу вам одним из первых. Если будут подходящие планировки, отправлю подборку и поясню, какие варианты лучше под вашу задачу.
```

### 8.2. Ипотечная заявка

Цель звонка — понять платёжеспособность и подготовить клиента к покупке.

Ключевые вопросы:

1. Сколько готовы внести первоначально?
2. Какой платёж комфортен?
3. Есть ли дети, подходящие под семейную ипотеку?
4. Есть ли материнский капитал?
5. Официальный доход или другой формат подтверждения?
6. Есть ли действующие кредиты?
7. Нужно ли продать свою недвижимость?

Финал:

```text
Сейчас задача — заранее понять ваш реальный бюджет, чтобы на старте продаж не тратить время и сразу смотреть подходящие квартиры.
```

### 8.3. Подбор квартиры

Цель звонка — получить критерии подбора и перевести клиента в понятный следующий шаг.

Ключевые вопросы:

1. Какая квартира нужна по комнатности?
2. Минимальная и комфортная площадь?
3. Максимальный бюджет?
4. Что важнее: цена, этаж, планировка, срок сдачи, ипотека?
5. Есть ли альтернативы: вторичка, другие дома, другой район?

Финал:

```text
Я подготовлю варианты и объясню, чем они отличаются по цене, планировке и условиям покупки.
```

### 8.4. Консультация

Цель звонка — ответить на вопрос и не потерять потенциального покупателя.

Правило: после ответа всегда переводить в следующий шаг.

```text
Я отвечу на ваш вопрос, а потом уточню пару параметров, чтобы понять, какая информация будет для вас полезной дальше.
```

### 8.5. Звонок

Цель — быстро связаться, не перегружая клиента.

```text
Здравствуйте, [Имя]. Вы заказали звонок на сайте по новостройкам Борисоглебска. Подскажите, какой вопрос для вас сейчас главный?
```

### 8.6. Список ожидания

Цель — зафиксировать ожидание и подготовить клиента к будущему касанию.

```text
Я добавил вас в список ожидания. Уточню, чтобы сообщить именно то, что вам важно: вас больше интересуют цены, планировки, ипотека или появление конкретной квартиры?
```

## 9. Юридически безопасная фиксация клиента

Фиксация должна быть не агрессивной, а доказательной и корректной.

Что фиксируем:

- дата и время заявки;
- имя и телефон;
- страница заявки;
- тип формы;
- ЖК или интерес;
- рекламные метки;
- текст согласия;
- ID фиксации;
- менеджер, который обработал заявку;
- история касаний;
- результат звонка.

Что нельзя писать клиенту:

- «мы вас забронировали», если брони нет;
- «цена зафиксирована», если цена не подтверждена;
- «вы закреплены за нами навсегда»;
- «застройщик подтвердил», если подтверждения нет.

Безопасные формулировки:

- «зафиксировали ваше обращение»;
- «зафиксировали ваш интерес к квартире/ЖК»;
- «добавили в список ожидания»;
- «сообщим после появления подтверждённых данных»;
- «предварительная заявка не является бронью и не фиксирует цену».

## 10. Следующие технические шаги

1. Подключить Supabase Edge Function или CRM webhook в `LEAD_ENDPOINT`.
2. Создать таблицу заявок по схеме из `docs/supabase-leads.sql`.
3. Настроить уведомления менеджеру.
4. Добавить антиспам: honeypot, rate limit, server-side validation.
5. Добавить страницу «Спасибо» и события аналитики.
6. Добавить call-tracking или отдельные номера под рекламные каналы.
7. Сделать мини-дашборд: заявки по источникам, формам, менеджерам, статусам.
