# Недельный отчёт по заявкам

## Назначение

Инструмент `/tools/weekly-report/` нужен для ручной еженедельной сводки после запуска заявок.

Он помогает быстро понять:

- сколько заявок пришло;
- сколько клиентов удалось дозвонить;
- сколько заявок горячие;
- сколько заявок связано с ипотекой;
- сколько клиентов сначала должны продать свою недвижимость;
- какой источник дал лучший результат;
- какие вопросы чаще всего задают покупатели.

## Адрес

```text
https://deputat36.github.io/bm/tools/weekly-report/
```

Страница закрыта от индексации через `noindex, nofollow`.

## Как использовать

1. Раз в неделю открыть инструмент.
2. Ввести данные из Web3Forms, таблицы, CRM или ручного учёта.
3. Сформировать текст отчёта.
4. Скопировать отчёт в чат отдела, Google Документ или задачу GitHub.
5. На основе отчёта принять решение по рекламе и контенту.

## Какие решения принимать

Если много заявок, но мало дозвона:

- проверить скорость обработки;
- проверить качество телефонов;
- добавить SMS/мессенджер после заявки.

Если много вопросов по ценам:

- усилить страницу `/ceny/`;
- подготовить пост о том, почему цены пока не опубликованы;
- собрать список клиентов на уведомление.

Если много ипотечных заявок:

- вести трафик на `/ipoteka/`;
- использовать ипотечный калькулятор;
- подключить ипотечного специалиста.

Если много клиентов с продажей своей недвижимости:

- вести трафик на `/pokupka-s-prodazhey/`;
- готовить оценку объектов;
- назначить ответственного риэлтора.
