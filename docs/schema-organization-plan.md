# План подключения Organization

`assets/js/schema.js` уже поддерживает генерацию JSON-LD типа `Organization`.

## 1. Где использовать

Основная страница-кандидат:

```text
/zastroyschiki/bm-group/
```

Цель: аккуратно описать карточку застройщика внутри городского портала, не создавая впечатление официального сайта застройщика.

## 2. Минимальное подключение

На страницу застройщика нужно добавить data-атрибут на `body`:

```html
<body data-schema-organization="BM Group Development">
```

Внизу страницы нужно подключить общий helper:

```html
<script src="../../assets/js/schema.js"></script>
```

После этого скрипт сформирует:

```text
Organization
```

## 3. Что проверить перед подключением

Перед подключением на страницу проверить:

- корректное публичное название застройщика;
- юридическое лицо, если оно будет выводиться на странице;
- видимый текст страницы;
- отсутствие формулировки «официальный сайт застройщика»;
- отсутствие неподтверждённых контактов;
- отсутствие обещаний цены, брони или наличия квартир.

## 4. Что не добавлять на текущем этапе

Пока не добавлять:

- контакты организации;
- логотип организации;
- внешние официальные ссылки;
- адреса офисов;
- коммерческие предложения;
- `Offer` и цены.

Это лучше добавить позже после проверки источников и юридической безопасности.

## 5. Проверка после подключения

В браузере выполнить:

```js
Array.from(document.querySelectorAll('script[type="application/ld+json"]')).map((item) => JSON.parse(item.textContent));
```

Ожидаемые типы на странице застройщика:

```text
BreadcrumbList
Organization
```

Если страница остаётся черновой и закрытой от индексации, микроразметку можно проверять технически, но не добавлять страницу в sitemap.
