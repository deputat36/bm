# План подключения ItemList

`assets/js/schema.js` поддерживает генерацию JSON-LD типа `ItemList` для страниц каталога.

## 1. Где использовать

Основные страницы-кандидаты:

```text
/novostroyki/
/zhk/
```

Цель: показать поисковым системам, что страница является списком объектов, а не произвольным набором блоков.

## 2. Минимальное подключение

На странице каталога добавить атрибут на `body`:

```html
<body data-schema-item-list="Новостройки Борисоглебска">
```

Каждую карточку объекта пометить:

```html
<article class="card" data-schema-list-item data-schema-item-name="ЖК Теллерманов сад" data-schema-item-url="../zhk/tellermanov-sad/">
```

Внизу страницы подключить helper:

```html
<script src="../assets/js/schema.js"></script>
```

После этого скрипт сформирует:

```text
ItemList
```

## 3. Как helper находит данные

Для каждой карточки он пытается взять:

1. `data-schema-item-name`;
2. заголовок `h2` или `h3`;
3. текст ссылки;
4. запасное название вида «Объект 1».

Ссылку он берёт из:

1. `data-schema-item-url`;
2. первой ссылки внутри карточки;
3. текущего URL страницы.

## 4. Что проверить перед подключением

Перед подключением проверить:

- карточки реально видны на странице;
- название в микроразметке совпадает с названием карточки;
- ссылка ведёт на карточку объекта;
- объект не является рабочей гипотезой;
- нет неподтверждённых цен;
- нет обещания наличия квартир.

## 5. Что не добавлять

Для каталога пока не добавлять:

- цены;
- `Offer`;
- количество доступных квартир;
- статус «в продаже», если это не подтверждено;
- условия покупки, которые могут измениться.

## 6. Проверка после подключения

В браузере выполнить:

```js
Array.from(document.querySelectorAll('script[type="application/ld+json"]')).map((item) => JSON.parse(item.textContent));
```

На странице каталога ожидаемые типы:

```text
BreadcrumbList
ItemList
```

## 7. Когда открывать для индексации

Открывать страницу с `ItemList` для индексации можно только после проверки:

- canonical;
- sitemap;
- всех ссылок карточек;
- юридически безопасных формулировок;
- соответствия JSON-LD видимому содержимому.
