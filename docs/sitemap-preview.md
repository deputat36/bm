# Предпросмотр sitemap портала

Дата фиксации: 2026-07-07

## 1. Назначение

Текущий файл `sitemap.xml` относится к старому сайту ЖК «Теллерманов сад» на домене `tellermanovsad.ru`. Его нельзя автоматически перезаписывать до завершения миграции.

Для безопасной подготовки нового sitemap добавлен инструмент предпросмотра:

```text
tools/build-sitemap-preview.mjs
```

Он выводит XML в консоль и не изменяет файлы проекта.

## 2. Запуск

```bash
npm run sitemap:preview
```

По умолчанию используется домен:

```text
https://novostroyki-borisoglebsk.ru
```

Можно указать другой домен:

```bash
PORTAL_BASE_URL=https://example.ru npm run sitemap:preview
```

Можно указать дату `lastmod`:

```bash
SITEMAP_LASTMOD=2026-07-07 npm run sitemap:preview
```

## 3. Источник данных

Инструмент читает:

```text
data/pages/index.json
```

## 4. Какие страницы попадают в sitemap

В sitemap попадают только страницы, у которых одновременно выполнено:

```text
status = published
robots не равен noindex,follow
```

Это защищает от случайной публикации черновых страниц.

## 5. Что будет сейчас

На текущем этапе новые страницы портала имеют статус:

```text
status = draft
robots = noindex,follow
```

Поэтому предпросмотр sitemap может вывести пустой `urlset`. Это нормальное поведение до момента публикации.

## 6. Как использовать перед запуском портала

Перед заменой боевого `sitemap.xml` нужно:

1. Проверить страницы вручную.
2. Запустить `npm run validate`.
3. Запустить `npm run urls:inventory`.
4. Перевести готовые страницы в `status = published`.
5. Убедиться, что у опубликованных страниц снят `noindex`.
6. Запустить `npm run sitemap:preview`.
7. Только после этого переносить результат в `sitemap.xml`.

## 7. Важное правило

Нельзя включать в sitemap страницы, по которым:

- не проверены данные из официальных источников;
- не проверены формы заявок;
- есть неподтвержденные цены, наличие квартир или условия покупки;
- не решены canonical и редиректы;
- страница всё ещё находится в статусе `draft` или `ready`.
