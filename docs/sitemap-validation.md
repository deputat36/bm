# Проверка текущего sitemap.xml

Дата фиксации: 2026-07-07

## 1. Назначение

Текущий `sitemap.xml` относится к старой структуре сайта ЖК «Теллерманов сад». До миграции его нельзя автоматически заменять, но нужно контролировать, чтобы в него случайно не попали черновые страницы нового портала.

Для этого в `tools/validate-static-site.mjs` добавлена проверка sitemap.

## 2. Что проверяется

Команда:

```bash
npm run validate
```

теперь дополнительно читает:

```text
sitemap.xml
```

и проверяет все значения внутри тегов:

```xml
<loc>...</loc>
```

## 3. Какие ситуации считаются предупреждениями

Валидатор выдаёт предупреждение, если:

- в sitemap есть дублирующийся `loc`;
- `loc` не является корректным URL;
- в sitemap попал URL чернового портального раздела;
- URL есть в `data/pages/index.json`, но статус страницы не `published`;
- URL есть в `data/pages/index.json`, но robots указан как `noindex,follow`;
- HTML-файл страницы существует и содержит `noindex`.

## 4. Почему это пока предупреждения, а не ошибки

Сейчас `sitemap.xml` ещё обслуживает старый сайт на `tellermanovsad.ru`.

На переходном этапе часть старых страниц может оставаться в sitemap, даже если для них уже подготовлены новые URL внутри `/zhk/tellermanov-sad/`.

Поэтому проверка работает в режиме предупреждений, чтобы:

- не сломать текущую индексацию старого сайта;
- не блокировать разработку портала;
- заранее подсветить рискованные URL перед миграцией.

## 5. Когда предупреждения нужно будет ужесточить

После запуска портала и выбора доменной стратегии предупреждения по sitemap можно перевести в ошибки для следующих случаев:

- sitemap содержит страницу со статусом `draft`;
- sitemap содержит страницу со статусом `ready`;
- sitemap содержит страницу с `robots = noindex,follow`;
- sitemap содержит черновые разделы `/portal-preview/`, `/zhk/`, `/novostroyki/`, `/sravnenie/`, `/spravochnik/`, `/zastroyschiki/` до снятия noindex.

## 6. Связанные инструменты

Для подготовки нового sitemap используются:

```text
npm run urls:inventory
npm run sitemap:preview
npm run validate
```

`urls:inventory` показывает, какие URL можно включать в sitemap.

`sitemap:preview` выводит будущий XML только из страниц со статусом `published` и без `noindex`.

`validate` проверяет, что текущий `sitemap.xml` не содержит очевидно опасных URL.
