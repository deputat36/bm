# Карта URL и план миграции в городской портал

Дата: 05.07.2026  
Проект: `novostroyki-borisoglebsk.ru`  
Репозиторий: `deputat36/bm`  
Базовая идея: текущий сайт ЖК «Теллерманов сад» не удаляется по смыслу, а становится одним из разделов городского портала по новостройкам Борисоглебска.

## 1. Цель документа

Этот документ нужен как практическая карта для разработки, SEO и контента.

Он отвечает на вопросы:

- какие страницы должны быть у портала;
- какие URL считаются основными;
- куда переносить текущие страницы сайта ЖК;
- какие страницы индексировать сразу;
- какие страницы создавать позже;
- где нужны canonical, 301-редиректы и обновление sitemap;
- какие title, description и H1 использовать на первом этапе.

## 2. Главный принцип архитектуры

Портал строится не вокруг одного объекта, а вокруг пользовательского спроса.

Правильная структура:

```text
/
/novostroyki/
/zhk/tellermanov-sad/
/kvartiry-v-novostroykakh/
/ceny/
/planirovki/
/ipoteka/
/zastroyschiki/
/rayony/
/guides/
/blog/
```

Неправильная структура:

```text
/o-zhk/
/kvartiry/
/planirovki/
/ceny/
/ipoteka/
```

Такая структура подходит сайту одного ЖК, но не городскому порталу. Для портала страницы `/ceny/`, `/planirovki/`, `/ipoteka/` должны стать городскими хабами, а страницы конкретного ЖК должны находиться внутри `/zhk/tellermanov-sad/`.

## 3. Правила URL

### 3.1. Общие правила

- Все URL — на латинице.
- Все URL — строчными буквами.
- Разделитель слов — дефис.
- В конце URL — слеш.
- Не использовать `.html` в новых публичных URL.
- Не смешивать транслитерацию и английские слова без необходимости.
- Не создавать страницы, если на них нет уникального полезного содержания.

### 3.2. Основные шаблоны

```text
/novostroyki/
/zhk/{zhk-slug}/
/zhk/{zhk-slug}/kvartiry/
/zhk/{zhk-slug}/planirovki/
```

## 5. Миграция текущих страниц

Текущий сайт ориентирован на один ЖК. При переходе к порталу текущие URL нужно разделить на 3 группы:

1. Страницы, которые становятся городскими хабами.
2. Страницы, которые переезжают внутрь `/zhk/tellermanov-sad/`.
3. Служебные/лидогенерационные страницы, которые не должны индексироваться.

### 5.3. Служебные страницы и лид-страницы

Сейчас в `robots.txt` уже закрыты многие служебные страницы: `/tools/`, `/docs/`, `/data/`, `/links/`, `/utm/`, `/realtor/`, `/qr/`, `/start/`, `/zayavka/`, `/callback/`, `/spisok-ozhidaniya/` и ряд посадочных страниц.

Для портала нужно сохранить принцип:

- служебные страницы закрывать от индексации;
- страницы заявок не индексировать;
- страницы с UTM не индексировать;
- страницы для конкретного риэлтора не индексировать;
- технические документы проекта не индексировать.

Но важно: коммерческие страницы, которые могут давать SEO-трафик, не нужно закрывать в robots, если они будут наполнены полноценным контентом.

## 6. Canonical и редиректы

### 6.1. Если домен `tellermanovsad.ru` остается отдельным

Вариант мягкой связки:

- `tellermanovsad.ru` остается лендингом ЖК;
- `novostroyki-borisoglebsk.ru/zhk/tellermanov-sad/` становится расширенной карточкой объекта;
- canonical на `tellermanovsad.ru` можно оставить на себя;
- на обоих сайтах должна быть взаимная ссылка;
- нужно избегать полного дубля текста.

Минус: два домена будут конкурировать по одним брендовым запросам.

### 6.2. Если главным становится `novostroyki-borisoglebsk.ru`

Вариант SEO-концентрации:

- `tellermanovsad.ru` делает 301 на `/zhk/tellermanov-sad/`;
- все старые разделы ЖК редиректятся на новые URL внутри портала;
- canonical везде указывает на `novostroyki-borisoglebsk.ru`;
- весь ссылочный вес и поведенческие сигналы собираются на одном домене.

Это более сильный вариант для городского портала, но требует аккуратного решения с позиционированием и юридическими формулировками.

### 6.3. Рекомендация

На первом этапе можно сделать так:

1. Не ломать текущий `tellermanovsad.ru` до запуска портала.
2. Подготовить портал в ветке/на тестовом домене.
3. После готовности портала решить: оставлять ли отдельный домен ЖК или переводить его в раздел портала.
4. Если выбираем единый портал — настроить 301 и обновить sitemap.

## 13. Что нельзя делать

- Нельзя оставлять `/ceny/` одновременно как страницу одного ЖК и как городскую страницу без четкого разделения.
- Нельзя делать одинаковый текст на `tellermanovsad.ru` и `novostroyki-borisoglebsk.ru/zhk/tellermanov-sad/`.
- Нельзя плодить страницы по типам квартир без данных.
- Нельзя писать «официальный сайт ЖК», если портал юридически не является официальным сайтом застройщика.
- Нельзя обещать цены, скидки, ипотечные условия и сроки без подтверждения.
- Нельзя закрывать от индексации основные коммерческие страницы, если они уже готовы и наполнены.
- Нельзя забывать дату обновления на страницах ЖК, цен, документов и хода строительства.

5. Обновить `sitemap.xml` и `robots.txt` под новый домен.
6. Проверить canonical и внутренние ссылки.
7. Добавить микроразметку `Organization`, `BreadcrumbList`, `FAQPage`, `Residence/ApartmentComplex`.
