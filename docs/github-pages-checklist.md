# Чек-лист запуска GitHub Pages

Репозиторий:

```text
deputat36/bm
```

Ожидаемый адрес:

```text
https://deputat36.github.io/bm/
```

## Настройка

1. Открыть репозиторий на GitHub.
2. Перейти в `Settings -> Pages`.
3. В поле `Source` выбрать `GitHub Actions`.
4. Сохранить настройку.
5. Открыть вкладку `Actions`.
6. Запустить или дождаться workflow `Deploy GitHub Pages`.
7. Проверить, что workflow завершился без ошибок.
8. Открыть `https://deputat36.github.io/bm/`.

## Проверка после запуска

1. Главная открывается без 404.
2. CSS и JS загружаются.
3. Логотип BM Group отображается.
4. Открываются страницы `/kvartiry/`, `/ceny/`, `/planirovki/`, `/contacts/`.
5. Форма заявки работает в локальном режиме.
6. `robots.txt` открывается.
7. `sitemap.xml` открывается.

## Что подключить сразу после запуска

- Яндекс.Метрика.
- Яндекс.Вебмастер.
- Google Search Console.
- Цель Метрики на отправку формы.
- Обработчик заявок в `assets/js/main.js`.
- UTM-ссылки для каждого риэлтора.

## Что не писать на сайте без подтверждения

- `официальный сайт ЖК`;
- `официальный представитель застройщика`;
- точные цены;
- точные площади;
- сроки сдачи;
- гарантированные скидки;
- обещание бронирования.

Корректная позиция: информационный сайт агентства недвижимости с предварительными заявками на консультацию.
