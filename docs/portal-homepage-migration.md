# План переноса главной страницы на портал

Сейчас корневой `index.html` остаётся рабочей главной ЖК «Теллерманов сад». Черновик будущей главной портала находится здесь:

```text
/portal-preview/
```

Такой подход нужен, чтобы не сломать текущий опубликованный сайт, рекламные ссылки, заявки и поисковый трафик.

## 1. Текущая ситуация

Текущий корень сайта:

```text
/
```

Фактически работает как лендинг первого объекта — ЖК «Теллерманов сад» / дом на Просторной 4А.

Новая карточка этого объекта внутри будущего портала:

```text
/zhk/tellermanov-sad/
```

Черновая портальная главная:

```text
/portal-preview/
```

## 2. Когда можно переносить `/portal-preview/` в `/`

Перенос можно делать только после выполнения условий:

1. Проверены все относительные пути в `/portal-preview/`.
2. Проверены ссылки на `/novostroyki/`, `/zhk/`, `/sravnenie/`, `/spravochnik/`, `/zastroyschiki/`, `/ipoteka/`.
3. Проверена форма заявки на подбор.
4. Проверено, что заявка попадает в email или Supabase после подключения endpoint.
5. Проверено, что после заявки открывается `/spasibo/`.
6. Пройден чек-лист `docs/lead-launch-checklist.md`.
7. Карточка ЖК `/zhk/tellermanov-sad/` готова принять трафик со старого корня.
8. Подготовлены canonical и sitemap.
9. Подготовлена карта редиректов.
10. Принято решение, что сайт теперь позиционируется как городской портал, а не как отдельный лендинг ЖК.

## 3. Что сделать перед переносом

### 3.1. Сохранить текущий лендинг ЖК

Перед заменой корневого `index.html` сохранить текущую главную ЖК как отдельную страницу, например:

```text
/zhk/tellermanov-sad/prostornaya-4a/
```

или усилить текущую карточку:

```text
/zhk/tellermanov-sad/
```

Важно: старый контент не удалять, потому что он содержит SEO-тексты, структуру, заявки и переходы.

### 3.2. Проверить разделы портала

Минимально должны быть готовы:

```text
/novostroyki/
/zhk/
/zhk/tellermanov-sad/
/zastroyschiki/
/zastroyschiki/bm-group/
/sravnenie/
/spravochnik/
/zayavka/
/spasibo/
/privacy/
```

### 3.3. Проверить формы

На главной портала должна быть форма:

```text
lead_type=portal_selection
form_id=portal_preview_home_selection
```

Проверить, что в заявке сохраняется:

- имя;
- телефон;
- тип заявки;
- ID формы;
- страница отправки;
- UTM;
- первое и последнее касание;
- согласие;
- квалификация;
- ID фиксации.

## 4. Порядок переноса

1. Скопировать содержимое `/portal-preview/index.html` в корневой `index.html`.
2. Изменить пути с `../assets/...` на `assets/...`.
3. Изменить ссылки с `../novostroyki/` на `novostroyki/`, с `../zhk/` на `zhk/` и так далее.
4. Убрать `noindex,follow` из новой главной только после финальной проверки.
5. Добавить canonical:

```html
<link rel="canonical" href="https://tellermanovsad.ru/">
```

6. Обновить Open Graph для портала.
7. Проверить все кнопки и форму.
8. Обновить `sitemap.xml`.
9. Проверить старые URL страниц ЖК.
10. После проверки удалить или оставить `/portal-preview/` закрытым от индексации.

## 5. Что нельзя делать

Нельзя резко заменить корневую главную, если:

- не проверены формы;
- не готова карточка ЖК;
- не готова карта URL;
- не проверены canonical;
- не ясно, куда вести рекламный трафик;
- нет понимания, какие страницы должны индексироваться.

## 6. Безопасное позиционирование новой главной

На главной портала использовать формулировки:

- «городской каталог новостроек»;
- «подбор квартиры»;
- «консультация специалиста»;
- «информация уточняется»;
- «данные сверяются по публичным источникам и предоставленным материалам».

Не использовать:

- «официальный сайт ЖК»;
- «официальный сайт застройщика»;
- «бронирование квартиры»;
- «фиксация цены»;
- «гарантированное одобрение ипотеки».

## 7. Решение после переноса

После переноса корень сайта должен отвечать на вопрос:

```text
Что такое «Новостройки Борисоглебска»?
```

Ответ:

```text
Это городской портал по новостройкам Борисоглебска, где покупатель может сравнить объекты, изучить ЖК, проверить документы, узнать про ипотеку и оставить заявку на подбор квартиры.
```
