# Правила публикации данных по новостройкам

Дата фиксации: 2026-07-07

## 1. Назначение

Документ фиксирует правила, по которым рабочие данные о новостройках переходят в публичные страницы портала «Новостройки Борисоглебска».

Главная задача — не вывести на сайт, в sitemap, рекламу или SEO-тексты сведения, которые ещё не проверены по официальным или письменно предоставленным источникам.

## 2. Основное правило публичного вывода

Объект можно использовать в публичной витрине только при одновременном выполнении условий:

```text
verification_status = confirmed
is_public_ready = true
is_active = true
```

`is_active = true` означает только то, что объект используется внутри проекта. Это не означает, что он уже готов к публичной индексации.

## 3. Статусы проверки

```text
confirmed            — есть официальный публичный источник, дата проверки и непротиворечивые данные
partially_confirmed  — часть данных подтверждена, но нужна повторная сверка или не хватает характеристик
requires_check       — данные нельзя публиковать как факт до проверки
do_not_publish       — данные нельзя выводить на публичный сайт, в рекламу, sitemap и SEO-тексты
```

## 4. Какие файлы должны содержать поля проверки

Поля проверки нужны минимум в следующих файлах:

```text
/data/projects/index.json
/data/projects/{slug}.json
/data/research/newbuilds-borisoglebsk-2018.json
```

Обязательные поля:

```text
verification_status
is_public_ready
last_checked_at
verification_note
```

## 5. Порядок добавления нового объекта

1. Сначала объект добавляется в рабочий реестр `data/research/newbuilds-borisoglebsk-2018.json`.
2. Для объекта фиксируются источники, дата проверки и статус.
3. Если данных недостаточно, объект остаётся в рабочем реестре и не переносится в публичные индексы.
4. После проверки создаётся подробная карточка `data/projects/{slug}.json`.
5. Затем объект добавляется в `data/projects/index.json`.
6. Если нужен новый застройщик, сначала создаётся `data/builders/{slug}.json`, затем обновляется `data/builders/index.json`.

## 6. Что нельзя публиковать без отдельной проверки

Нельзя выводить как факт без актуального подтверждения:

- цены;
- наличие квартир;
- скидки;
- акции;
- ипотечные ставки;
- обещания одобрения ипотеки;
- сроки передачи ключей, если они отличаются от документов;
- планы строительства, которые не подтверждены публичными документами;
- закрытые рабочие сведения, которые застройщик просил не публиковать.

## 7. Правило для страниц и индексации

Если страница ещё не прошла проверку, она должна оставаться в режиме:

```text
meta robots = noindex,follow
pages/index.json: robots = noindex,follow
pages/index.json: status = draft или ready
```

В sitemap попадают только страницы со статусом `published`.

## 8. Правило для менеджеров и рекламы

Заявка с сайта не должна формулироваться как бронь квартиры, фиксация цены или гарантированное одобрение ипотеки.

Безопасная формулировка:

```text
Оставьте заявку, чтобы получить консультацию по новостройкам Борисоглебска и доступным вариантам покупки.
```

Небезопасная формулировка:

```text
Забронируйте квартиру по текущей цене и получите гарантированное одобрение ипотеки.
```

## 9. Текущий статус ЖК «Теллерманов сад»

В основных JSON-файлах объект помечен как:

```text
verification_status = partially_confirmed
is_public_ready = false
```

Это означает, что данные можно использовать как рабочую основу проекта, но перед снятием `noindex`, включением в sitemap и рекламным продвижением нужна повторная сверка с ЕИСЖС / наш.дом.рф и проектной декларацией.
