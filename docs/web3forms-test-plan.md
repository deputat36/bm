# План тестирования Web3Forms

## Цель

Проверить отправку заявок через Web3Forms до подключения ключа в боевые формы сайта.

## Тестовая страница

После публикации GitHub Pages:

```text
https://deputat36.github.io/bm/web3forms-test/
```

Страница закрыта от индексации через `noindex, nofollow`.

## Порядок проверки

1. Создать `access_key` в Web3Forms.
2. Открыть `/web3forms-test/`.
3. Вставить ключ в поле `Web3Forms access_key`.
4. Нажать `Отправить тест`.
5. Проверить, что заявка появилась в Web3Forms.
6. Проверить, что уведомление пришло на почту, указанную в Web3Forms.
7. Если тест прошёл, вставить ключ в `assets/js/main.js`.
8. Выполнить боевую проверку:

```bash
REQUIRE_WEB3FORMS=1 node scripts/check-site.js
```

9. Отправить тестовую заявку с главной страницы.
10. Отправить тестовую заявку со страницы `/contacts/`.

## Что должно прийти в заявке

- имя;
- телефон;
- интерес;
- комментарий;
- страница отправки;
- заголовок страницы;
- referrer;
- UTM-метки;
- дата отправки.

## Что делать при ошибке

1. Проверить, что ключ вставлен без пробелов.
2. Проверить статус формы в Web3Forms.
3. Проверить лимиты тарифа.
4. Проверить, не попала ли заявка в спам.
5. Повторить тест с другой почтой получателя.

## После успешного запуска

1. Добавить цели в Яндекс.Метрику.
2. Проверить страницу `/spasibo/`.
3. Проверить UTM-ссылки риэлторов.
4. Сделать одну тестовую заявку с `realtor=test`.
5. Зафиксировать регламент обработки заявок.
