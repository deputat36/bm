# Внутренняя перелинковка

Цель перелинковки - распределить вес по страницам, помочь поисковикам быстрее найти все разделы и вести посетителя к заявке.

## Главная

Главная должна ссылаться на:

- `/dom-prostornaya-4a/`
- `/kvartiry/`
- `/planirovki/`
- `/ceny/`
- `/ipoteka/`
- `/hod-stroitelstva/`
- `/contacts/`

Анкоры должны быть естественными:

- `узнать о доме`
- `выбрать квартиру`
- `получить планировки`
- `узнать цены`
- `оставить заявку`

## Страница квартир

`/kvartiry/` должна ссылаться на:

- `/kvartiry/1-komnatnye/`
- `/kvartiry/2-komnatnye/`
- `/kvartiry/3-komnatnye/`
- `/planirovki/`
- `/ceny/`
- `/contacts/`

Главная задача страницы - быстро развести посетителя по типу квартиры и привести к форме.

## Квартирные страницы

На каждой странице 1-, 2- и 3-комнатных квартир нужны ссылки на:

- соседние типы квартир;
- страницу планировок;
- страницу цен;
- страницу ипотеки;
- форму заявки.

Пример логики:

```text
1-комнатные -> цены -> ипотека -> заявка
2-комнатные -> планировки -> инфраструктура -> заявка
3-комнатные -> дом -> закрытая территория -> заявка
```

## Новости и статьи

Каждая статья должна иметь:

- 2-4 внутренние ссылки в тексте;
- один блок `Читайте также`;
- один CTA-блок на заявку;
- ссылку обратно в `/novosti/`.

Приоритетные связки:

- статья про кирпичный дом -> `/dom-prostornaya-4a/`;
- статья про предчистовую отделку -> `/planirovki/` и `/kvartiry/`;
- статья про ипотеку -> `/ipoteka/` и `/contacts/`;
- статья про закрытую территорию -> `/infrastruktura/` и `/dom-prostornaya-4a/`.

## Частотность обновления

После каждого нового материала нужно проверить:

1. Есть ли ссылка на него из `/novosti/`.
2. Есть ли минимум 2 ссылки из старых страниц на новый материал.
3. Ведет ли новый материал к заявке.
4. Обновлен ли `sitemap.xml`.

## Правило для SEO

Не использовать одинаковый анкор `подробнее` везде. Лучше писать конкретно:

- `цены на квартиры на Просторной 4А`;
- `планировки квартир`;
- `ипотечная консультация`;
- `заявка на квартиру`;
- `ход строительства дома`.
