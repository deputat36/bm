# Подключение Web3Forms

## Что выбрано

Для первого этапа сбора заявок используем Web3Forms. Это подходит для GitHub Pages, потому что сайту не нужен собственный сервер.

Заявки будут уходить из текущих HTML-форм через `assets/js/main.js`.

## Что нужно от владельца проекта

1. Зарегистрироваться на Web3Forms.
2. Создать `access_key`.
3. Передать ключ для вставки в `assets/js/main.js`.

В коде нужно заполнить:

```js
WEB3FORMS_ACCESS_KEY: "ваш_ключ"
```

## Какие данные отправляются

- имя;
- телефон;
- интерес;
- способ покупки;
- комментарий;
- страница отправки;
- заголовок страницы;
- referrer;
- дата и время;
- `utm_source`;
- `utm_medium`;
- `utm_campaign`;
- `utm_content`;
- `utm_term`;
- `realtor`;
- `realtor_id`.

## Как работает сейчас

Если `WEB3FORMS_ACCESS_KEY` пустой, заявки сохраняются только локально в браузере в `localStorage`. Это режим разработки.

Если ключ заполнен, заявка отправляется в Web3Forms, а пользователь перенаправляется на `/spasibo/`.

## Supabase

Supabase на этом этапе не обязателен. Его можно подключить позже как второй слой:

- хранить все заявки в таблице;
- делать статусы обработки;
- смотреть отчёты по риэлторам;
- дублировать заявки из Web3Forms через webhook/automation.

Первый запуск проще сделать через Web3Forms, а Supabase добавить после проверки спроса.
