# Цели Яндекс.Метрики и аналитики

## Основные цели

| Цель | Событие | Где использовать |
| --- | --- | --- |
| Отправка заявки | `lead_submit` | Все формы на сайте |
| Ошибка заявки | `lead_error` | Проверка проблем с обработчиком |
| Клик по форме | `lead_form_focus` | Анализ интереса без отправки |
| Переход в контакты | `contacts_open` | Главная, статьи, посадочные страницы |
| Клик по телефону | `phone_click` | После добавления реального телефона |
| Переход по UTM риэлтора | сегмент `realtor` | Оценка работы каждого специалиста |

## Что добавить в код после создания счётчика

В `assets/js/main.js` после успешной отправки заявки нужно вызывать цель:

```js
if (window.ym && SITE_CONFIG.metrikaId) {
  ym(SITE_CONFIG.metrikaId, "reachGoal", "lead_submit");
}
```

После ошибки:

```js
if (window.ym && SITE_CONFIG.metrikaId) {
  ym(SITE_CONFIG.metrikaId, "reachGoal", "lead_error");
}
```

## Сегменты

Нужно создать сегменты:

- посетители с `utm_campaign = prostornaya_start`;
- посетители с `utm_campaign = prostornaya_kvartiry`;
- посетители с `utm_campaign = prostornaya_mortgage`;
- посетители с параметром `realtor`;
- посетители страниц `/investoram/`;
- посетители страниц `/semeyam/`;
- посетители страниц `/pokupka-s-prodazhey/`;
- посетители страницы `/dokumenty/`.

## Отчёты

Минимальный набор отчётов:

1. Источники трафика.
2. Страницы входа.
3. Конверсии по формам.
4. Конверсии по UTM-кампаниям.
5. Конверсии по риэлторам.
6. Отказы по посадочным страницам.

## Что считать хорошим результатом

На раннем этапе важны не только заявки, но и признаки спроса:

- переходы из VK на сайт;
- дочитывание статей;
- переходы в `/contacts/`;
- повторные визиты;
- сохранение UTM и `realtor` в заявке;
- вопросы по планировкам, ценам, ипотеке и документам.
