# Настройка домена novostroyki-borisoglebsk.ru

## Главная рекомендация

Домен `novostroyki-borisoglebsk.ru` лучше подключать не к подпапке текущего репозитория, а к отдельному репозиторию.

Рекомендуемый репозиторий:

`deputat36/novostroyki-borisoglebsk`

Почему:

- GitHub Pages подключает домен к корню сайта, а не к отдельной папке внутри другого проекта;
- текущий репозиторий `bm` связан с временным сайтом ЖК;
- новый домен должен быть самостоятельным городским сайтом;
- так проще не смешивать `tellermanovsad.ru` и `novostroyki-borisoglebsk.ru`.

## Что сделать в GitHub

1. Создать новый репозиторий:

`deputat36/novostroyki-borisoglebsk`

2. Перенести содержимое папки:

`novostroyki-borisoglebsk/`

в корень нового репозитория.

3. В корне нового репозитория должен быть файл:

`CNAME`

С содержимым:

`novostroyki-borisoglebsk.ru`

4. В настройках репозитория открыть:

Settings → Pages

5. Выбрать:

- Source: Deploy from a branch
- Branch: main
- Folder: /root

6. В Custom domain указать:

`novostroyki-borisoglebsk.ru`

7. После проверки DNS включить:

Enforce HTTPS

## Что прописать у регистратора домена

Для корневого домена `novostroyki-borisoglebsk.ru` создать A-записи:

- `@` → `185.199.108.153`
- `@` → `185.199.109.153`
- `@` → `185.199.110.153`
- `@` → `185.199.111.153`

Для `www` создать CNAME:

- `www` → `deputat36.github.io.`

Дополнительно можно добавить AAAA-записи IPv6:

- `@` → `2606:50c0:8000::153`
- `@` → `2606:50c0:8001::153`
- `@` → `2606:50c0:8002::153`
- `@` → `2606:50c0:8003::153`

## Что удалить у регистратора

Если у домена уже стоят парковочные записи регистратора, их нужно удалить или заменить:

- старые A-записи;
- старые CNAME;
- URL Redirect;
- парковочные записи;
- wildcard `*`, если он есть.

## Проверка

После настройки DNS подождать от 10 минут до 24 часов.

Проверить:

- открывается ли `https://novostroyki-borisoglebsk.ru/`;
- открывается ли `https://www.novostroyki-borisoglebsk.ru/`;
- включился ли HTTPS;
- нет ли ошибки Custom domain в GitHub Pages.

## Важный момент

До запуска нового репозитория не нужно менять CNAME текущего сайта `tellermanovsad.ru`, чтобы не сломать временный сайт ЖК.
