# Модель данных каталога новостроек

## Назначение

Единый формат карточки проекта нужен, чтобы все новостройки на сайте описывались одинаково и без рекламных преувеличений.

## Поля проекта

- `id` — технический идентификатор;
- `name` — название проекта;
- `slug` — адрес страницы;
- `city` — город;
- `address` — адрес;
- `status` — статус проекта;
- `class` — класс жилья, если подтвержден;
- `floors` — этажность;
- `wall_material` — материал стен;
- `apartments_total` — количество квартир;
- `apartment_types` — типы квартир;
- `area_min` — минимальная площадь;
- `area_max` — максимальная площадь;
- `deadline` — срок сдачи;
- `keys_deadline` — срок передачи ключей;
- `documents` — документы и ссылки;
- `source_notes` — откуда взята информация;
- `lead_interest_value` — значение для формы заявки.

## Правила заполнения

1. Указывать только проверяемые данные.
2. Если данных нет, писать «уточняется», а не придумывать.
3. Не использовать формулировки официального сайта.
4. Не обещать бронь, цену или условия без подтверждения.
5. Отдельно хранить источник информации.

## Статусы проекта

- `draft` — заготовка;
- `published` — публичная карточка;
- `needs_review` — требует проверки;
- `archived` — скрыто из каталога.

## Поля для заявок

Минимум для качественной заявки:

- имя;
- телефон;
- интересующий проект;
- тип квартиры;
- бюджет или способ покупки;
- удобное время связи.
